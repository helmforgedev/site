# Kafka

<span data-i18n="docsCharts.kafka.intro">
  Deploy Apache Kafka on Kubernetes using the official `apache/kafka:4.2.0` image with a KRaft-only design. The chart
  intentionally supports a simple `single-broker` path for development and a production-oriented `cluster` path with
  dedicated controllers and brokers.
</span>

## <span data-i18n="docsCharts.heading.keyFeatures">Key Features</span>

- <span data-i18n="docsCharts.kafka.feature.latestStable">
    Latest stable Kafka — pinned to the latest stable upstream Kafka release
  </span>
- <span data-i18n="docsCharts.kafka.feature.kraftOnly">KRaft only — no ZooKeeper mode in this chart</span>
- <span data-i18n="docsCharts.kafka.feature.twoTopologies">
    Two clear topologies — `single-broker` for simple environments and `cluster` for production-oriented deployments
  </span>
- <span data-i18n="docsCharts.kafka.feature.persistentStorage">
    Persistent storage — supported for single-broker, controllers, and brokers
  </span>
- <span data-i18n="docsCharts.kafka.feature.stableDns">
    Stable broker DNS — advertised listeners use StatefulSet pod DNS names inside the cluster
  </span>
- <span data-i18n="docsCharts.kafka.feature.optionalMetrics">
    Optional metrics — JMX exporter javaagent with optional ServiceMonitor
  </span>

## <span data-i18n="docsCharts.heading.installation">Installation</span>

**<span data-i18n="docsCharts.install.httpsRepo">HTTPS repository:</span>**

```bash
helm repo add helmforge https://repo.helmforge.dev
helm repo update
helm install kafka helmforge/kafka -f values.yaml
```

**<span data-i18n="docsCharts.install.ociRegistry">OCI registry:</span>**

```bash
helm install kafka oci://ghcr.io/helmforgedev/helm/kafka -f values.yaml
```

## <span data-i18n="docsCharts.heading.singleBrokerExample">Single-Broker Example</span>

```yaml
architecture: single-broker

singleBroker:
  persistence:
    size: 8Gi
```

## <span data-i18n="docsCharts.heading.clusterExample">Cluster Example</span>

```yaml
architecture: cluster

cluster:
  minInSyncReplicas: 2
  controllers:
    replicaCount: 3
    persistence:
      size: 8Gi
  brokers:
    replicaCount: 3
    persistence:
      size: 50Gi

pdb:
  enabled: true
```

## <span data-i18n="docsCharts.heading.validationGuidance">Validation Guidance</span>

- <span data-i18n="docsCharts.kafka.note.singleBrokerDev">
    Use `single-broker` for development and CI, not as the production baseline
  </span>
- <span data-i18n="docsCharts.kafka.note.clusterProduction">
    Use `cluster` for production-oriented workloads with dedicated controllers and brokers
  </span>
- <span data-i18n="docsCharts.kafka.note.validateFlows">
    Validate topic creation, producer, and consumer flows before promotion
  </span>
- <span data-i18n="docsCharts.kafka.note.externalListeners">
    Plan external listeners, TLS, and auth as explicit follow-up work if your environment needs them
  </span>
