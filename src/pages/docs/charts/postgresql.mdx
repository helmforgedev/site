# PostgreSQL

<span data-i18n="docsCharts.postgresql.intro">
  Production-ready PostgreSQL deployment with support for standalone and streaming replication architectures.
</span>

## <span data-i18n="docsCharts.heading.keyFeatures">Key Features</span>

- <span data-i18n="docsCharts.postgresql.feature.standaloneReplication">
    <strong>Standalone and replication</strong> — Single instance or primary with streaming replicas
  </span>
- <span data-i18n="docsCharts.postgresql.feature.automaticInit">
    <strong>Automatic initialization</strong> — Init scripts, extensions, and custom configuration
  </span>
- <span data-i18n="docsCharts.postgresql.feature.s3Backup">
    <strong>S3 backup</strong> — Scheduled backups to S3-compatible storage via CronJob
  </span>
- <span data-i18n="docsCharts.postgresql.feature.metrics">
    <strong>Metrics</strong> — Prometheus exporter with ServiceMonitor
  </span>
- <span data-i18n="docsCharts.postgresql.feature.security">
    <strong>Security</strong> — Non-root containers, network policies, TLS support
  </span>
- <span data-i18n="docsCharts.postgresql.feature.storage">
    <strong>Persistent storage</strong> — Configurable PVCs with storage class selection
  </span>

## <span data-i18n="docsCharts.heading.installation">Installation</span>

```bash
helm repo add helmforge https://repo.helmforge.dev
helm repo update
helm install my-pg helmforge/postgresql
```

<div data-tab-panel="standalone">

```yaml
# values.yaml
architecture: standalone

auth:
  postgresPassword: 'my-secret-password'
  database: myapp
  username: myuser
  password: 'user-password'

primary:
  persistence:
    size: 20Gi

metrics:
  enabled: true
  serviceMonitor:
    enabled: true
```

</div>

<div data-tab-panel="replication">

```yaml
# values.yaml
architecture: replication

auth:
  postgresPassword: 'my-secret-password'
  database: myapp
  username: myuser
  password: 'user-password'
  replicationPassword: 'repl-password'

primary:
  persistence:
    size: 20Gi

readReplicas:
  replicaCount: 2
  persistence:
    size: 20Gi
```

</div>

<div data-tab-panel="backup">

```yaml
# values.yaml
architecture: standalone

auth:
  postgresPassword: 'my-secret-password'
  database: myapp

primary:
  persistence:
    size: 20Gi

backup:
  enabled: true
  schedule: '0 2 * * *'
  s3:
    endpoint:
      https://s3.amazonaws.com
    bucket: my-pg-backups
    accessKey: AKIAIOSFODNN7EXAMPLE
    secretKey: wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY
  retention:
    days: 30
```

</div>

<div data-tab-panel="ingress">

```yaml
# values.yaml — Use with pgAdmin or a TCP ingress controller
architecture: standalone

auth:
  postgresPassword: 'my-secret-password'
  database: myapp

primary:
  persistence:
    size: 20Gi

# For pgAdmin or web-based management tools
ingress:
  enabled: true
  ingressClassName: traefik
  hosts:
    - host: pgadmin.example.com
      paths:
        - path: /
          pathType: Prefix
  tls:
    - secretName: pgadmin-tls
      hosts:
        - pgadmin.example.com
  # annotations:
  #   cert-manager.io/cluster-issuer: letsencrypt-prod
```

</div>

## <span data-i18n="docsCharts.heading.commonIssues">Common Issues</span>

<Callout type="danger" title="Pod stuck in CrashLoopBackOff">
  <span data-i18n="docsCharts.postgresql.issue.crashloop">
    Usually caused by incorrect `auth.postgresPassword` on upgrade. PostgreSQL does not re-initialize
    auth when the data directory already exists. Check logs with `kubectl logs <pod>` and verify the
    password matches the existing data.
  </span>
</Callout>

<Callout type="tip" title="Slow replication sync">
  <span data-i18n="docsCharts.postgresql.issue.replicationSync">
    For large databases, initial replica sync can take time. Increase `readReplicas.resources` and ensure the storage
    class provides adequate IOPS. Monitor replication lag with the Prometheus exporter (`pg_replication_lag` metric).
  </span>
</Callout>

<Callout type="info" title="Using with connection poolers">
  <span data-i18n="docsCharts.postgresql.issue.connectionPoolers">
    For high-connection workloads, deploy PgBouncer alongside PostgreSQL. Point your application to PgBouncer and
    configure it to connect to the PostgreSQL service.
  </span>
</Callout>
