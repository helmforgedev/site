# AdGuard Home

Deploy [AdGuard Home](https://adguard.com/adguard-home/overview.html) on Kubernetes using the official [adguard/adguardhome](https://hub.docker.com/r/adguard/adguardhome) Docker image. Network-wide DNS-level ad and tracker blocking for all devices.

## <span data-i18n="docsCharts.heading.keyFeatures">Key Features</span>
- <span data-i18n="docsCharts.auto.adguard_home.b0">Network-wide ad blocking — DNS-level filtering for all devices</span>
- <span data-i18n="docsCharts.auto.adguard_home.b1">Two deployment modes — wizard (port 3000) or pre-configured (port 80)</span>
- <span data-i18n="docsCharts.auto.adguard_home.b2">DNS over HTTPS / TLS — encrypted upstream DNS resolvers</span>
- <span data-i18n="docsCharts.auto.adguard_home.b3">AdGuardHome Sync — multi-instance synchronization via adguardhome-sync</span>
- <span data-i18n="docsCharts.auto.adguard_home.b4">S3 backup — automated CronJob-based backup to any S3-compatible storage</span>
- <span data-i18n="docsCharts.auto.adguard_home.b5">Separate DNS service — dedicated LoadBalancer for DNS traffic</span>
- <span data-i18n="docsCharts.auto.adguard_home.b6">Ingress support — TLS with cert-manager for the web UI</span>
- <span data-i18n="docsCharts.auto.adguard_home.b7">Config seeding — initial configuration preserved across upgrades</span>

## <span data-i18n="docsCharts.heading.installation">Installation</span>

**<span data-i18n="docsCharts.install.httpsRepo">HTTPS repository:</span>**

```bash
helm repo add helmforge https://repo.helmforge.dev
helm repo update
helm install adguard-home helmforge/adguard-home
```

**<span data-i18n="docsCharts.install.ociRegistry">OCI registry:</span>**

```bash
helm install adguard-home oci://ghcr.io/helmforgedev/helm/adguard-home
```

## <span data-i18n="docsCharts.heading.wizardModeDefault">Wizard Mode (Default)</span>

By default, AdGuard Home starts in wizard mode on port 3000. Complete the setup via the web UI:

```bash
helm install adguard-home oci://ghcr.io/helmforgedev/helm/adguard-home \
  --set service.dns.type=LoadBalancer \
  --set service.dns.loadBalancerIP=192.168.1.53
kubectl port-forward svc/adguard-home-adguard-home-web 3000:80
# Visit http://localhost:3000
```

## <span data-i18n="docsCharts.heading.preConfiguredMode">Pre-Configured Mode</span>

Skip the wizard by providing `config.adGuardHome`:

```yaml
config:
adGuardHome:
  http:
address: 0.0.0.0:80
users:
  - name: admin # bcrypt hash — generate
with: htpasswd -bnBC 10 "" 'PASSWORD' | cut
-d: -f2
password: '$2y$10$...'
dns:
  bind_hosts:
  - 0.0.0.0
port: 53
upstream_dns:
  - https://dns.cloudflare.com/dns-query
  - https://dns.google/dns-query
bootstrap_dns:
  - 1.1.1.1
protection_enabled: true
filtering_enabled: true
filters:
  - enabled: true
url:
  https://adguardteam.github.io/HostlistsRegistry/assets/filter_1.txt
name: AdGuard DNS filter
id: 1
schema_version: 29
service:
dns:
  type: LoadBalancer
loadBalancerIP: '192.168.1.53'
```

## <span data-i18n="docsCharts.heading.multiInstanceSync">Multi-Instance Sync</span>

Synchronize configuration from an origin to replicas using adguardhome-sync:

```yaml
sync:
enabled: true
origin:
  url: 'http://adguard-primary:80'
username: admin
password: changeme
replicas:
  - url: 'http://adguard-replica:80'
username: admin
password: changeme
```
