# Appwrite

<span data-i18n="docsCharts.appwrite.intro.before">Helm chart for deploying</span> [Appwrite](https://appwrite.io/)
<span data-i18n="docsCharts.appwrite.intro.after">
  on Kubernetes — a self-hosted Backend-as-a-Service (BaaS) platform providing authentication, databases, storage,
  functions, messaging, and a real-time API. Ships with bundled MariaDB and Redis by default.
</span>

## <span data-i18n="docsCharts.heading.keyFeatures">Key Features</span>

- <span data-i18n="docsCharts.appwrite.feature.fullStack">
    Full Appwrite stack — API, Console (v7.5.7), Realtime, 12 workers, schedulers, and maintenance
  </span>
- <span data-i18n="docsCharts.appwrite.feature.bundledMariadb">
    Bundled MariaDB — default install includes the HelmForge MariaDB chart
  </span>
- <span data-i18n="docsCharts.appwrite.feature.bundledRedis">
    Bundled Redis — default install includes the HelmForge Redis chart
  </span>
- <span data-i18n="docsCharts.appwrite.feature.externalServices">
    External services support — connect to managed MariaDB/MySQL and Redis instead of subcharts
  </span>
- <span data-i18n="docsCharts.appwrite.feature.pathIngress">
    Path-based ingress routing — single host with path prefixes for API (`/v1`), Console, and Realtime (`/v1/realtime`)
  </span>
- <span data-i18n="docsCharts.appwrite.feature.sharedPvcs">
    Shared PVCs — persistent volumes for uploads, cache, and function builds shared across components
  </span>
- <span data-i18n="docsCharts.appwrite.feature.appVersion">
    appVersion 1.8.1 — tracks the latest Appwrite open-source release
  </span>

## <span data-i18n="docsCharts.heading.installation">Installation</span>

**<span data-i18n="docsCharts.install.httpsRepo">HTTPS repository:</span>**

```bash
helm repo add helmforge https://repo.helmforge.dev
helm repo update
helm install appwrite helmforge/appwrite
```

**<span data-i18n="docsCharts.install.ociRegistry">OCI registry:</span>**

```bash
helm install appwrite oci://ghcr.io/helmforgedev/helm/appwrite
```

## <span data-i18n="docsCharts.heading.basicExample">Basic Example</span>

```yaml
# values.yaml
appwrite:
  domain: appwrite.example.com
  secretKey: 'change-me-to-a-random-string'

ingress:
  enabled: true
  ingressClassName: traefik
  hosts:
    - host: appwrite.example.com
      paths:
        - path: /
          pathType: Prefix

mariadb:
  enabled: true

redis:
  enabled: true
```

## <span data-i18n="docsCharts.heading.externalDatabaseAndCache">External Database and Cache</span>

```yaml
mariadb:
  enabled: false

redis:
  enabled: false

database:
  external:
    host: mariadb.example.internal
    port: 3306
    name: appwrite
    existingSecret: appwrite-db

cache:
  external:
    host: redis.example.internal
    port: 6379
    existingSecret: appwrite-redis
```

## <span data-i18n="docsCharts.heading.operationalNotes">Operational Notes</span>

- <span data-i18n="docsCharts.appwrite.note.alphaWorkloads">
    This alpha chart deploys approximately 20 Kubernetes workloads — use minimal worker replicas for initial testing
  </span>
- <span data-i18n="docsCharts.appwrite.note.paths">
    The Console is served at the root path, the API at `/v1`, and Realtime at `/v1/realtime`
  </span>
- <span data-i18n="docsCharts.appwrite.note.defaultBundled">
    Default installs use bundled MariaDB and Redis via HelmForge subcharts
  </span>
- <span data-i18n="docsCharts.appwrite.note.sameImage">
    All worker pods share the same Appwrite image with different entrypoint commands
  </span>
