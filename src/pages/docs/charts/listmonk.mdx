# Listmonk

<span data-i18n="docsCharts.listmonk.intro.before">Helm chart for deploying</span> [Listmonk](https://listmonk.app/)
<span data-i18n="docsCharts.listmonk.intro.after">
  on Kubernetes — a self-hosted newsletter and mailing list manager with bundled PostgreSQL, persistent uploads storage,
  and optional S3 backup.
</span>

## <span data-i18n="docsCharts.heading.keyFeatures">Key Features</span>

- <span data-i18n="docsCharts.listmonk.feature.officialImage">
    Official Listmonk image based on `listmonk/listmonk`
  </span>
- <span data-i18n="docsCharts.listmonk.feature.bundledPostgresql">
    Bundled PostgreSQL — default install includes the HelmForge PostgreSQL chart
  </span>
- <span data-i18n="docsCharts.listmonk.feature.externalDatabase">
    External database support — managed PostgreSQL can be used instead of the subchart
  </span>
- <span data-i18n="docsCharts.listmonk.feature.persistentUploads">
    Persistent uploads — configurable PVC for media and attachment storage
  </span>
- <span data-i18n="docsCharts.listmonk.feature.s3Backup">
    S3 backup — optional CronJob for PostgreSQL dumps to S3-compatible storage
  </span>
- <span data-i18n="docsCharts.listmonk.feature.idempotentBootstrap">
    Idempotent bootstrap — init container runs `--install --idempotent --yes` and `--upgrade --yes` on every start
  </span>
- <span data-i18n="docsCharts.listmonk.feature.pgcrypto">
    pgcrypto extension — automatically provisioned via PostgreSQL init scripts
  </span>

## <span data-i18n="docsCharts.heading.installation">Installation</span>

**<span data-i18n="docsCharts.install.httpsRepo">HTTPS repository:</span>**

```bash
helm repo add helmforge https://repo.helmforge.dev
helm repo update
helm install listmonk helmforge/listmonk
```

**<span data-i18n="docsCharts.install.ociRegistry">OCI registry:</span>**

```bash
helm install listmonk oci://ghcr.io/helmforgedev/helm/listmonk
```

## <span data-i18n="docsCharts.heading.basicExample">Basic Example</span>

```yaml
listmonk:
  adminUser: admin
  adminPassword: changeme

ingress:
  enabled: true
  ingressClassName: traefik
  hosts:
    - host: listmonk.example.com
      paths:
        - path: /
          pathType: Prefix
```

## <span data-i18n="docsCharts.heading.externalDatabase">External Database</span>

```yaml
database:
  mode: external
  external:
    host: postgres.example.internal
    name: listmonk
    username: listmonk
    existingSecret: listmonk-db

postgresql:
  enabled: false
```

## <span data-i18n="docsCharts.heading.withS3Backup">With S3 Backup</span>

```yaml
backup:
  enabled: true
  schedule: '0 3 * * *'
  s3:
    endpoint:
      https://minio.example.internal
    bucket: listmonk-backups
    existingSecret: listmonk-s3
```

## <span data-i18n="docsCharts.heading.operationalNotes">Operational Notes</span>

- <span data-i18n="docsCharts.listmonk.note.smtpConfig">
    SMTP configuration is done through the Listmonk admin UI after first install
  </span>
- <span data-i18n="docsCharts.listmonk.note.initContainer">
    The init container ensures the database schema is created and migrated on every pod start
  </span>
- <span data-i18n="docsCharts.listmonk.note.pgcryptoRequired">
    PostgreSQL requires the `pgcrypto` extension, which the chart provisions automatically via init scripts
  </span>
