# Cloudflare Tunnel (cloudflared)

Deploy Cloudflare Tunnel on Kubernetes using the official [cloudflare/cloudflared](https://hub.docker.com/r/cloudflare/cloudflared) Docker image. Secure, outbound-only connections between your cluster and Cloudflare's network — no open ports, no public IP required.

## <span data-i18n="docsCharts.heading.keyFeatures">Key Features</span>

- <span data-i18n="docsCharts.auto.cloudflared.b0">Zero-trust networking — no inbound firewall rules needed</span>
- <span data-i18n="docsCharts.auto.cloudflared.b1">
    Remotely-managed — configure routes in the Cloudflare dashboard
  </span>
- <span data-i18n="docsCharts.auto.cloudflared.b2">
    High availability — 2 replicas with PodDisruptionBudget by default
  </span>
- <span data-i18n="docsCharts.auto.cloudflared.b3">Prometheus metrics — `/ready` and `/metrics` on port 2000</span>
- <span data-i18n="docsCharts.auto.cloudflared.b4">ServiceMonitor — optional Prometheus Operator integration</span>
- <span data-i18n="docsCharts.auto.cloudflared.b5">Existing secrets — bring your own Secret for the tunnel token</span>

## <span data-i18n="docsCharts.heading.installation">Installation</span>

**<span data-i18n="docsCharts.install.httpsRepo">HTTPS repository:</span>**

```bash
helm repo add helmforge https://repo.helmforge.dev
helm repo update
helm install cloudflared helmforge/cloudflared -f values.yaml
```

**<span data-i18n="docsCharts.install.ociRegistry">OCI registry:</span>**

```bash
helm install cloudflared oci://ghcr.io/helmforgedev/helm/cloudflared -f values.yaml
```

## <span data-i18n="docsCharts.heading.basicExample">Basic Example</span>

```yaml
# values.yaml
tunnel:
token: 'eyJhIjoiY2Y...' # From Cloudflare dashboard
```

Create a tunnel in the [Cloudflare dashboard](https://one.dash.cloudflare.com) under **Networks → Tunnels**, copy the token, and configure public hostnames to route traffic to your Kubernetes services.

## <span data-i18n="docsCharts.heading.productionExample">Production Example</span>

```yaml
tunnel:
  existingSecret: cloudflare-tunnel
replicaCount: 2
pdb:
  enabled: true
  minAvailable: 1
resources:
  requests:
    cpu: 50m
    memory: 64Mi
  limits:
    memory: 128Mi
serviceMonitor:
  enabled: true
labels:
  release: prometheus
```

## <span data-i18n="docsCharts.heading.importantNotes">Important Notes</span>

- <span data-i18n="docsCharts.auto.cloudflared.b6">
    Do not use HPA — downscaling terminates active tunnel connections
  </span>
- <span data-i18n="docsCharts.auto.cloudflared.b7">
    No ingress template — cloudflared replaces traditional ingress controllers
  </span>
- <span data-i18n="docsCharts.auto.cloudflared.b8">
    Routing is dashboard-managed — use the Cloudflare dashboard to map hostnames to services
  </span>
