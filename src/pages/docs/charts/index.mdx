## <span data-i18n="docsCharts.index.commonFeatures">Common Features</span><span data-i18n="docsCharts.index.commonFeaturesIntro">All HelmForge charts share these capabilities:</span>
- <span data-i18n="docsCharts.auto.index.b0">Multiple architectures — Standalone, replicated, or clustered modes where applicable</span>
- <span data-i18n="docsCharts.auto.index.b1">Security hardening — Network policies, pod security contexts, non-root containers</span>
- <span data-i18n="docsCharts.auto.index.b2">Observability — Prometheus metrics, ServiceMonitor, and alerting rules</span>
- <span data-i18n="docsCharts.auto.index.b3">Ingress support — Configurable ingress with `ingressClassName` and TLS</span>
- <span data-i18n="docsCharts.auto.index.b4">Resource management — CPU/memory requests and limits with sensible defaults</span>
- <span data-i18n="docsCharts.auto.index.b5">Pod Disruption Budgets — Configurable PDBs for high-availability deployments</span>
