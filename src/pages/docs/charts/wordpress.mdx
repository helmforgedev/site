# WordPress

Deploy WordPress on Kubernetes using the official [wordpress](https://hub.docker.com/_/wordpress)
Docker image (Apache variant). Supports MySQL subchart or external MySQL/MariaDB, scheduled backups
of both the database and `wp-content`, PHP ini tuning, and Prometheus metrics via an Apache exporter
sidecar.

<Callout type="warning" title="Default PHP upload limit is 2 MB — increase before uploading media">
  The official WordPress Apache image ships with `upload_max_filesize = 2M` and `post_max_size = 8M`. Any image, video,
  or plugin upload over 2 MB will fail silently or with a generic error. Configure `php.ini` via the chart to raise
  these limits before installing themes or uploading content.
</Callout>

## Key Features

- **Official Apache image** — PHP + Apache, compatible with all WordPress plugins and themes
- **MySQL subchart** — bundled MySQL or external MySQL/MariaDB via `database.external`
- **`configExtra`** — inject arbitrary PHP into `wp-config.php` (`WP_MEMORY_LIMIT`, cron, etc.)
- **PHP ini tuning** — custom `php.ini` via ConfigMap for upload limits and memory
- **Dual S3 backup** — mysqldump (database) + tar (wp-content) in a single scheduled CronJob
- **Apache exporter** — Prometheus metrics via sidecar with optional ServiceMonitor
- **Ingress with TLS** — cert-manager integration

## Installation

**HTTPS repository:**

```bash
helm repo add helmforge https://repo.helmforge.dev
helm repo update
helm install wordpress helmforge/wordpress -f values.yaml
```

**OCI registry:**

```bash
helm install wordpress oci://ghcr.io/helmforgedev/helm/wordpress -f values.yaml
```

<div data-tab-panel="basic">

```yaml
# values.yaml — WordPress with bundled MySQL, PHP tuning, and TLS
wordpress:
  siteTitle: 'My Blog'
  adminUser: admin
  adminPassword: 'change-me'
  adminEmail: admin@example.com

  # Inject extra PHP into wp-config.php
  configExtra: |
    define('WP_MEMORY_LIMIT', '256M');
    define('WP_POST_REVISIONS', 5);
    define('DISABLE_WP_CRON', true);

# Custom PHP ini — raise upload limits (default 2M is too low)
php:
  ini: |
    upload_max_filesize = 64M
    post_max_size = 64M
    memory_limit = 256M
    max_execution_time = 300

mysql:
  enabled: true
  auth:
    password: 'db-password'

persistence:
  enabled: true
  size: 10Gi

ingress:
  enabled: true
  ingressClassName: traefik
  annotations:
    cert-manager.io/cluster-issuer: letsencrypt-prod
  hosts:
    - host: blog.example.com
      paths:
        - path: /
          pathType: Prefix
  tls:
    - secretName: wordpress-tls
      hosts:
        - blog.example.com
```

wordpress:

    mysqldumpArgs: '--single-transaction --quick --skip-lock-tables --no-tablespaces'

| `wordpress.extraEnv`      | array   | `[]`                | Extra environment variables for the container.                                   |

<Callout type="tip" title="Use configExtra for WordPress constants">
  `wordpress.configExtra` is the cleanest way to set WordPress PHP constants without building a custom image. Common
  uses: `WP_MEMORY_LIMIT`, `DISABLE_WP_CRON` (use Kubernetes CronJob instead), `WP_POST_REVISIONS`, `WP_CACHE`, and
  `FORCE_SSL_ADMIN`.
</Callout>

### Admin Credentials

### Database

**Auto-detection precedence** (`database.mode: auto`):

1. `database.external.host` or `database.external.existingSecret` → external DB
2. `mysql.enabled: true` → MySQL subchart
3. No fallback — WordPress requires a database

| Parameter                                     | Type    | Default             | Description                                    |
| --------------------------------------------- | ------- | ------------------- | ---------------------------------------------- |
| `database.mode`                               | string  | `auto`              | Database mode: `auto`, `external`, or `mysql`. |
| `database.external.host`                      | string  | `""`                | External database hostname.                    |
| `database.external.port`                      | integer | `3306`              | External database port.                        |
| `database.external.name`                      | string  | `wordpress`         | Database name.                                 |
| `database.external.username`                  | string  | `wordpress`         | Database username.                             |
| `database.external.password`                  | string  | `""`                | Database password (prefer `existingSecret`).   |
| `database.external.existingSecret`            | string  | `""`                | Existing secret with database password.        |
| `database.external.existingSecretPasswordKey` | string  | `database-password` | Key inside the existing secret.                |

### PHP Configuration

<Callout type="warning" title="Default upload limit is 2 MB — too low for any real content">
  Always set `php.ini` upload limits before allowing editors to upload content. The Apache image defaults
  (`upload_max_filesize = 2M`, `post_max_size = 8M`) will block plugin installs, theme uploads, and media over 2 MB.
  Recommended minimums: `64M` upload, `64M` post, `256M` memory.
</Callout>

| Parameter | Type   | Default | Description                                                             |
| --------- | ------ | ------- | ----------------------------------------------------------------------- |
| `php.ini` | string | `""`    | Custom `php.ini` content injected via ConfigMap. Applied to Apache PHP. |

### Backup

The backup CronJob creates two archives per run and uploads both to S3:

- **Database**: `mysqldump` using `docker.io/library/mysql:8.4`
- **Files**: `tar` of `/var/www/html/wp-content` using Alpine

| Parameter                           | Type    | Default                            | Description                                                      |
| ----------------------------------- | ------- | ---------------------------------- | ---------------------------------------------------------------- |
| `backup.enabled`                    | boolean | `false`                            | Enable scheduled S3 backup CronJob.                              |
| `backup.schedule`                   | string  | `"0 3 * * *"`                      | Cron schedule.                                                   |
| `backup.suspend`                    | boolean | `false`                            | Suspend CronJob without deleting it.                             |
| `backup.concurrencyPolicy`          | string  | `Forbid`                           | CronJob concurrency policy.                                      |
| `backup.successfulJobsHistoryLimit` | integer | `3`                                | Successful job records to keep.                                  |
| `backup.failedJobsHistoryLimit`     | integer | `3`                                | Failed job records to keep.                                      |
| `backup.backoffLimit`               | integer | `1`                                | Job retry limit.                                                 |
| `backup.archivePrefix`              | string  | `wordpress`                        | Prefix for backup archive filenames.                             |
| `backup.images.worker`              | string  | `docker.io/library/mysql:8.4`      | Image providing `mysqldump`.                                     |
| `backup.images.archiver`            | string  | `docker.io/library/alpine:3.22`    | Image providing `tar` for wp-content archive.                    |
| `backup.images.uploader`            | string  | `docker.io/helmforge/mc:1.0.0`     | Image providing MinIO client for S3 upload.                      |
| `backup.s3.endpoint`                | string  | `""`                               | S3-compatible endpoint URL.                                      |
| `backup.s3.bucket`                  | string  | `""`                               | Target bucket name.                                              |
| `backup.s3.existingSecret`          | string  | `""`                               | Existing secret with S3 credentials.                             |
| `backup.database.host`              | string  | `""`                               | Override DB host for backup (inherits app credentials if empty). |
| `backup.database.mysqldumpArgs`     | string  | `--single-transaction --quick ...` | Extra arguments for `mysqldump`.                                 |
