# Uptime Kuma

<span data-i18n="docsCharts.uptimeKuma.intro.before">Deploy</span> [Uptime Kuma](https://uptime.kuma.pet)
<span data-i18n="docsCharts.uptimeKuma.intro.after">
  on Kubernetes using the official `louislam/uptime-kuma` Docker image. Self-hosted monitoring with HTTP/TCP/DNS/Ping
  checks, 90+ notification services, and customizable status pages.
</span>

## <span data-i18n="docsCharts.heading.keyFeatures">Key Features</span>

- <span data-i18n="docsCharts.uptimeKuma.feature.monitorTypes">
    20+ monitor types — HTTP(s), TCP, Ping, DNS, Docker, WebSocket, and more
  </span>
- <span data-i18n="docsCharts.uptimeKuma.feature.notificationServices">
    90+ notification services — Telegram, Discord, Slack, Email, Pushover, and more
  </span>
- <span data-i18n="docsCharts.uptimeKuma.feature.statusPages">
    Status pages — public status pages with custom domains
  </span>
- <span data-i18n="docsCharts.uptimeKuma.feature.sqliteMariaDb">
    SQLite or MariaDB — embedded SQLite (default) or MySQL subchart
  </span>
- <span data-i18n="docsCharts.uptimeKuma.feature.externalDatabase">
    External database — connect to existing MariaDB instances
  </span>
- <span data-i18n="docsCharts.uptimeKuma.feature.scheduledBackups">
    Scheduled backups — SQLite tar or mysqldump with S3 upload
  </span>
- <span data-i18n="docsCharts.uptimeKuma.feature.ingressSupport">Ingress support — TLS with cert-manager</span>
- <span data-i18n="docsCharts.uptimeKuma.feature.twoFa">2FA — built-in two-factor authentication</span>

## <span data-i18n="docsCharts.heading.installation">Installation</span>

**<span data-i18n="docsCharts.install.httpsRepo">HTTPS repository:</span>**

```bash
helm repo add helmforge https://repo.helmforge.dev
helm repo update
helm install uptime-kuma helmforge/uptime-kuma -f values.yaml
```

**<span data-i18n="docsCharts.install.ociRegistry">OCI registry:</span>**

```bash
helm install uptime-kuma oci://ghcr.io/helmforgedev/helm/uptime-kuma -f values.yaml
```

## <span data-i18n="docsCharts.heading.mariadbMode">MariaDB Mode</span>

```yaml
database:
  type: mariadb

mysql:
  enabled: true
  auth:
    password: 'change-me'
```

## <span data-i18n="docsCharts.heading.externalDatabase">External Database</span>

```yaml
database:
  type: mariadb
  external:
    host: mariadb.example.com
    name: uptime_kuma
    username: uptime_kuma
    existingSecret: uptime-kuma-db-credentials

mysql:
  enabled: false
```
