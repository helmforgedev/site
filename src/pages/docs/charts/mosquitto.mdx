# Mosquitto

Helm chart for deploying [Eclipse Mosquitto](https://mosquitto.org/) on Kubernetes using the official [`eclipse-mosquitto`](https://hub.docker.com/_/eclipse-mosquitto) Docker image, with support for standalone or federated broker topologies and an optional [`emqx/mqttx-web`](https://hub.docker.com/r/emqx/mqttx-web) companion UI.

## <span data-i18n="docsCharts.heading.keyFeatures">Key Features</span>

- <span data-i18n="docsCharts.auto.mosquitto.b0">Official Mosquitto image based on `eclipse-mosquitto`</span>
- <span data-i18n="docsCharts.auto.mosquitto.b1">
    Standalone mode single broker deployment for the simplest and most predictable runtime
  </span>
- <span data-i18n="docsCharts.auto.mosquitto.b2">
    Federated mode multiple bridged brokers using Mosquitto bridge connections across StatefulSet peers
  </span>
- <span data-i18n="docsCharts.auto.mosquitto.b3">
    WebSocket listener browser-ready MQTT access on a dedicated listener
  </span>
- <span data-i18n="docsCharts.auto.mosquitto.b4">
    MQTTX Web companion optional web client deployment with separate scaling controls
  </span>
- <span data-i18n="docsCharts.auto.mosquitto.b5">
    Authentication and ACL optional username/password and ACL file generation
  </span>
- <span data-i18n="docsCharts.auto.mosquitto.b6">
    Ingress support configurable ingress for broker WebSocket access and the optional MQTTX Web UI
  </span>

## <span data-i18n="docsCharts.heading.installation">Installation</span>

**HTTPS Repository**

```bash
helm repo add helmforge https://repo.helmforge.dev
helm repo update
helm install mosquitto helmforge/mosquitto
```

**OCI Registry**

```bash
helm install mosquitto oci://ghcr.io/helmforgedev/helm/mosquitto
```

## <span data-i18n="docsCharts.heading.basicExample">Basic Example</span>

```yaml
architecture:
mode:
  standalonebroker:
replicaCount: 1
service:
sessionAffinity: None
```

## <span data-i18n="docsCharts.heading.federatedExample">Federated Example</span>

```yaml
architecture:
mode:
  federatedbroker:
replicaCount: 3mqttxWeb:
enabled: true
replicaCount: 1
service:
sessionAffinity: None
```

## <span data-i18n="docsCharts.heading.topologyNotes">Topology Notes</span>

- `standalone` mode requires exactly one broker replica- `federated` mode requires at least two broker replicas- federated mode uses official Mosquitto bridge connections, not a native shared-state cluster- broker replicas do not share sessions or retained state like EMQX-style clustering- `mqttxWeb.replicaCount` is independent from `broker.replicaCount`
