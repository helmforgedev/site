# Open WebUI

<span data-i18n="docsCharts.openWebui.intro.before">Deploy</span> [Open WebUI](https://docs.openwebui.com/)
<span data-i18n="docsCharts.openWebui.intro.after">
  on Kubernetes — a self-hosted AI chat platform with Ollama and OpenAI-compatible backend support, RAG pipelines,
  multi-model conversations, and an extensible plugin system.
</span>

## <span data-i18n="docsCharts.heading.keyFeatures">Key Features</span>

- <span data-i18n="docsCharts.openWebui.feature.ollamaIntegration">
    Ollama integration — connect to local or remote Ollama instances for open-source LLMs
  </span>
- <span data-i18n="docsCharts.openWebui.feature.openaiCompatibleApi">
    OpenAI-compatible API — works with OpenAI, Azure, Anthropic proxies, LiteLLM, and more
  </span>
- <span data-i18n="docsCharts.openWebui.feature.sqlitePostgresql">
    SQLite or PostgreSQL — embedded SQLite for simple installs, PostgreSQL subchart for production
  </span>
- <span data-i18n="docsCharts.openWebui.feature.redisCoordination">
    Redis WebSocket coordination — required for multi-instance deployments
  </span>
- <span data-i18n="docsCharts.openWebui.feature.ragPipelines">
    RAG pipelines — retrieval-augmented generation with document uploads
  </span>
- <span data-i18n="docsCharts.openWebui.feature.s3Backup">
    S3 backup — scheduled PostgreSQL backup to S3-compatible storage
  </span>
- <span data-i18n="docsCharts.openWebui.feature.ingressSupport">
    Ingress support — TLS via cert-manager with configurable ingress class
  </span>

## <span data-i18n="docsCharts.heading.installation">Installation</span>

**<span data-i18n="docsCharts.install.httpsRepo">HTTPS repository:</span>**

```bash
helm repo add helmforge https://repo.helmforge.dev
helm repo update
helm install open-webui helmforge/open-webui
```

**<span data-i18n="docsCharts.install.ociRegistry">OCI registry:</span>**

```bash
helm install open-webui oci://ghcr.io/helmforgedev/helm/open-webui
```

## <span data-i18n="docsCharts.heading.basicExample">Basic Example</span>

```yaml
openWebui:
  ollamaBaseUrl: http://ollama.default.svc:11434

ingress:
  enabled: true
  ingressClassName: traefik
  hosts:
    - host: chat.example.com
      paths:
        - path: /
          pathType: Prefix
```

## <span data-i18n="docsCharts.heading.productionExample">Production Example</span>

```yaml
postgresql:
  enabled: true

redis:
  enabled: true

openWebui:
  ollamaBaseUrl: http://ollama.default.svc:11434
  openaiBaseUrl: https://api.openai.com/v1
  openaiExistingSecret: openai-credentials

persistence:
  size: 20Gi

ingress:
  enabled: true
  ingressClassName: traefik
  hosts:
    - host: chat.example.com
      paths:
        - path: /
          pathType: Prefix
  tls:
    - hosts:
        - chat.example.com
      secretName: open-webui-tls
```

## <span data-i18n="docsCharts.heading.operationalNotes">Operational Notes</span>

- <span data-i18n="docsCharts.openWebui.note.sqliteSingle">
    SQLite mode is single-instance only — no horizontal scaling
  </span>
- <span data-i18n="docsCharts.openWebui.note.postgresqlReplicas">
    PostgreSQL mode supports multiple replicas with Redis for WebSocket coordination
  </span>
- <span data-i18n="docsCharts.openWebui.note.autoSecret">
    The chart auto-generates a session secret key if not provided
  </span>
- <span data-i18n="docsCharts.openWebui.note.telemetryDisabled">
    Telemetry is disabled by default (`DO_NOT_TRACK=true`)
  </span>
