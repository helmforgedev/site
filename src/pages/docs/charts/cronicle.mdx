# Cronicle

Deploy Cronicle on Kubernetes using the [soulteary/cronicle](https://hub.docker.com/r/soulteary/cronicle) container image. Cronicle is a multi-server task scheduler and runner with a web-based UI for managing scheduled jobs, cron replacements, and automated workflows.

## <span data-i18n="docsCharts.heading.keyFeatures">Key Features</span>

- <span data-i18n="docsCharts.auto.cronicle.b0">
    Web UI — browser-based interface for creating and managing scheduled tasks
  </span>
- <span data-i18n="docsCharts.auto.cronicle.b1">
    Multi-Server — optional UDP auto-discovery for distributed job execution
  </span>
- <span data-i18n="docsCharts.auto.cronicle.b2">Job Storage — persistent storage for job data, history, and state</span>
- <span data-i18n="docsCharts.auto.cronicle.b3">
    Email Notifications — configurable SMTP for job success/failure alerts
  </span>
- <span data-i18n="docsCharts.auto.cronicle.b4">
    Session Encryption — auto-generated secret key for secure sessions
  </span>
- <span data-i18n="docsCharts.auto.cronicle.b5">
    Concurrency Control — configurable maximum concurrent jobs per server
  </span>

## <span data-i18n="docsCharts.heading.installation">Installation</span>

**<span data-i18n="docsCharts.install.httpsRepo">HTTPS repository:</span>**

```bash
helm repo add helmforge https://repo.helmforge.dev
helm repo update
helm install cronicle helmforge/cronicle
```

**<span data-i18n="docsCharts.install.ociRegistry">OCI registry:</span>**

```bash
helm install cronicle oci://ghcr.io/helmforgedev/helm/cronicle
```

## <span data-i18n="docsCharts.heading.basicExample">Basic Example</span>

```yaml
# values.yaml
cronicle:
baseUrl: 'https://cronicle.example.com'
maxJobs: 5
emailFrom: 'cronicle@example.com'
smtpHostname: 'smtp.example.com'
persistence:
enabled: true
size: 5Gi
ingress:
enabled: true
ingressClassName: traefik
hosts:
  - host: cronicle.example.com
paths:
  - path: /
pathType: Prefix
```
