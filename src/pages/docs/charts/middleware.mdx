# Middleware

Deploy [Middleware](https://www.middlewarehq.com/) on Kubernetes — an open-source DORA metrics platform that measures Deployment Frequency, Lead Time, MTTR, and Change Failure Rate for engineering teams.

## <span data-i18n="docsCharts.heading.keyFeatures">Key Features</span>

- <span data-i18n="docsCharts.auto.middleware.b0">
    All four DORA metrics — Deployment Frequency, Lead Time for Changes, MTTR, Change Failure Rate
  </span>
- <span data-i18n="docsCharts.auto.middleware.b1">
    All-in-one container — frontend, analytics, and sync server in a single image
  </span>
- <span data-i18n="docsCharts.auto.middleware.b2">PostgreSQL + Redis — bundled subcharts or external connections</span>
- <span data-i18n="docsCharts.auto.middleware.b3">
    CI/CD integrations — tracks metrics from your existing pipelines
  </span>
- <span data-i18n="docsCharts.auto.middleware.b4">Persistent storage — API keys and configuration stored in PVC</span>
- <span data-i18n="docsCharts.common.feature.ingressCertManager">
    **Ingress support** — TLS via cert-manager with configurable ingress class
  </span>

## <span data-i18n="docsCharts.heading.installation">Installation</span>

**<span data-i18n="docsCharts.install.httpsRepo">HTTPS repository:</span>**

```bash
helm repo add helmforge https://repo.helmforge.dev
helm repo update
helm install middleware helmforge/middleware -f values.yaml
```

**<span data-i18n="docsCharts.install.ociRegistry">OCI registry:</span>**

```bash
helm install middleware oci://ghcr.io/helmforgedev/helm/middleware -f values.yaml
```

## <span data-i18n="docsCharts.heading.basicExample">Basic Example</span>

```yaml
postgresql:
enabled: true
redis:
enabled: true
ingress:
enabled: true
ingressClassName: traefik
hosts:
  - host: middleware.example.com
paths:
  - path: /
pathType: Prefix
```

## <span data-i18n="docsCharts.heading.operationalNotes">Operational Notes</span>

- single instance only — no horizontal scaling- resource-heavy — 16GB RAM recommended for production workloads- all three ports (frontend, analytics, sync) are exposed via the same pod
