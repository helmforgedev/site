# ntfy

Deploy ntfy on Kubernetes using the official [binwiederhier/ntfy](https://hub.docker.com/r/binwiederhier/ntfy) container image. ntfy is a simple HTTP-based pub-sub notification service that lets you send push notifications to your phone or desktop via scripts, without signup or cost.

## <span data-i18n="docsCharts.heading.keyFeatures">Key Features</span>

- <span data-i18n="docsCharts.auto.ntfy.b0">
    REST API & WebSocket — publish and subscribe to topics via simple HTTP PUT/POST or WebSocket
  </span>
- <span data-i18n="docsCharts.auto.ntfy.b1">
    Firebase & UnifiedPush — supports Android push delivery through FCM and UnifiedPush
  </span>
- <span data-i18n="docsCharts.auto.ntfy.b2">
    Prometheus Metrics — optional `/metrics` endpoint with ServiceMonitor support
  </span>
- <span data-i18n="docsCharts.auto.ntfy.b3">
    Behind-Proxy Mode — trusts `X-Forwarded-For` headers for correct client identification
  </span>
- <span data-i18n="docsCharts.auto.ntfy.b4">
    Attachment Support — configurable file attachment size and expiry limits
  </span>
- <span data-i18n="docsCharts.auto.ntfy.b5">
    Persistence — persistent storage for cache and authentication databases
  </span>

## <span data-i18n="docsCharts.heading.installation">Installation</span>

**<span data-i18n="docsCharts.install.httpsRepo">HTTPS repository:</span>**

```bash
helm repo add helmforge https://repo.helmforge.dev
helm repo update
helm install ntfy helmforge/ntfy
```

**<span data-i18n="docsCharts.install.ociRegistry">OCI registry:</span>**

```bash
helm install ntfy oci://ghcr.io/helmforgedev/helm/ntfy
```

## <span data-i18n="docsCharts.heading.basicExample">Basic Example</span>

```yaml
# values.yaml
ntfy:
baseUrl: 'https://ntfy.example.com'
behindProxy: true
enableMetrics: true
persistence:
enabled: true
size: 2Gi
ingress:
enabled: true
ingressClassName: traefik
hosts:
  - host: ntfy.example.com
paths:
  - path: /
pathType: Prefix
```
