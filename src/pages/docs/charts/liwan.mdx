# Liwan

Ultra-lightweight, privacy-first web analytics with embedded DuckDB storage. Liwan runs as a single binary with
minimal resource requirements and no external database dependency — all analytics data lives in a local DuckDB file
on a persistent volume.

## Key Features

- **Single binary** — no external database, no sidecar, minimal footprint
- **Privacy-first** — no cookies, no cross-site tracking, GDPR-compliant by design
- **DuckDB storage** — columnar analytics engine embedded in the binary
- **GeoIP enrichment** — optional IP-to-location lookup stored alongside DuckDB data
- **Ingress support** — TLS via cert-manager with configurable ingress class
- **Persistent storage** — PVC-backed `/data` directory for DuckDB and GeoIP files

## Installation

**HTTPS repository:**

```bash
helm repo add helmforge https://repo.helmforge.dev
helm repo update
helm install liwan helmforge/liwan
```

**OCI registry:**

```bash
helm install liwan oci://ghcr.io/helmforgedev/helm/liwan
```

## Configuration Reference

### Core

| Parameter          | Type   | Default | Description                          |
| ------------------ | ------ | ------- | ------------------------------------ |
| `nameOverride`     | string | `""`    | Override the chart name.             |
| `fullnameOverride` | string | `""`    | Override the full release name.      |
| `commonLabels`     | object | `{}`    | Extra labels added to all resources. |

### Image

| Parameter          | Type   | Default                         | Description                          |
| ------------------ | ------ | ------------------------------- | ------------------------------------ |
| `image.repository` | string | `ghcr.io/explodingcamera/liwan` | Liwan container image.               |
| `image.tag`        | string | `"1.4.0"`                       | Image tag.                           |
| `image.pullPolicy` | string | `IfNotPresent`                  | Image pull policy.                   |
| `imagePullSecrets` | array  | `[]`                            | Pull secrets for private registries. |

### Liwan Configuration

| Parameter        | Type    | Default | Description                                    |
| ---------------- | ------- | ------- | ---------------------------------------------- |
| `liwan.port`     | integer | `9042`  | Application HTTP port inside the container.    |
| `liwan.baseUrl`  | string  | `""`    | Public base URL of the Liwan instance.         |
| `liwan.extraEnv` | array   | `[]`    | Extra environment variables for the container. |

<Callout type="info" title="baseUrl is required for link generation">
  Set `liwan.baseUrl` to your public URL (for example `https://analytics.example.com`). Without it, Liwan may generate
  incorrect self-referencing links in the UI.
</Callout>

## Common Issues

<Callout type="danger" title="Data loss on persistence.enabled: false">
  If `persistence.enabled` is set to `false`, all analytics data stored in DuckDB is lost when the pod restarts. Always
  enable persistence in production.
</Callout>

<Callout type="tip" title="Right-size your PVC from the start">
  DuckDB compresses analytics data efficiently. For most small-to-medium sites, `2Gi` is sufficient for years of data.
  PVC resize requires a StorageClass with `allowVolumeExpansion: true`.
</Callout>
