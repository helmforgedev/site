# Flowise

<span data-i18n="docsCharts.flowise.intro.before">Helm chart for deploying</span> [Flowise](https://flowiseai.com/)
<span data-i18n="docsCharts.flowise.intro.after">
  on Kubernetes using the official `flowiseai/flowise` image, with a simple standalone topology and a scalable queue
  topology for larger workloads.
</span>

## <span data-i18n="docsCharts.heading.keyFeatures">Key Features</span>

- <span data-i18n="docsCharts.flowise.feature.officialImage">Official Flowise image based on `flowiseai/flowise`</span>
- <span data-i18n="docsCharts.flowise.feature.standaloneMode">
    Standalone mode — SQLite and local storage for simple installs
  </span>
- <span data-i18n="docsCharts.flowise.feature.queueMode">
    Queue mode — separate main and worker deployments for scalable processing
  </span>
- <span data-i18n="docsCharts.flowise.feature.bundledDeps">
    Bundled PostgreSQL and Redis — optional subcharts for queue mode
  </span>
- <span data-i18n="docsCharts.flowise.feature.externalSql">External SQL support — external PostgreSQL or MySQL</span>
- <span data-i18n="docsCharts.flowise.feature.s3Storage">
    S3-backed shared storage — queue mode uses S3-compatible storage for shared blob data
  </span>
- <span data-i18n="docsCharts.flowise.feature.ingressSupport">
    Ingress support — configurable ingress for the Flowise web UI
  </span>

## <span data-i18n="docsCharts.heading.installation">Installation</span>

**<span data-i18n="docsCharts.install.httpsRepo">HTTPS repository:</span>**

```bash
helm repo add helmforge https://repo.helmforge.dev
helm repo update
helm install flowise helmforge/flowise
```

**<span data-i18n="docsCharts.install.ociRegistry">OCI registry:</span>**

```bash
helm install flowise oci://ghcr.io/helmforgedev/helm/flowise
```

## <span data-i18n="docsCharts.heading.standaloneExample">Standalone Example</span>

```yaml
architecture:
  mode: standalone

flowise:
  appUrl:
    https://flowise.example.com

ingress:
  enabled: true
  ingressClassName: traefik
  hosts:
    - host: flowise.example.com
      paths:
        - path: /
          pathType: Prefix
```

## <span data-i18n="docsCharts.heading.queueModeExample">Queue Mode Example</span>

```yaml
architecture:
  mode: queue

flowise:
  replicaCount: 2

queue:
  worker:
    replicaCount: 2

persistence:
  enabled: false

postgresql:
  enabled: true

redis:
  enabled: true

storage:
  type: s3
  s3:
    bucketName: flowise
    existingSecret: flowise-s3
```

## <span data-i18n="docsCharts.heading.topologyNotes">Topology Notes</span>

- <span data-i18n="docsCharts.flowise.note.standaloneDefaults">
    `standalone` defaults to SQLite plus a local PVC for the easiest install
  </span>
- <span data-i18n="docsCharts.flowise.note.queueRequirements">
    `queue` requires Redis, a SQL database, `storage.type=s3`, and `persistence.enabled=false`
  </span>
- <span data-i18n="docsCharts.flowise.note.queueScaleOut">
    Queue mode is intended for scale-out deployments with separate main and worker pods
  </span>
- <span data-i18n="docsCharts.flowise.note.localSingleNode">Local storage is intentionally single-node oriented</span>
