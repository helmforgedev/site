# MongoDB

<span data-i18n="docsCharts.mongodb.intro">
  Production-ready MongoDB deployment with support for standalone and replica set configurations.
</span>

## <span data-i18n="docsCharts.heading.keyFeatures">Key Features</span>

- <span data-i18n="docsCharts.mongodb.feature.standaloneReplicaset">
    <strong>Standalone and replica set</strong> — Single instance or multi-member replica set
  </span>
- <span data-i18n="docsCharts.mongodb.feature.automaticInit">
    <strong>Automatic initialization</strong> — Init scripts and custom configuration
  </span>
- <span data-i18n="docsCharts.mongodb.feature.metrics">
    <strong>Metrics</strong> — Prometheus exporter with ServiceMonitor
  </span>
- <span data-i18n="docsCharts.mongodb.feature.security">
    <strong>Security</strong> — Authentication, non-root containers, network policies
  </span>
- <span data-i18n="docsCharts.mongodb.feature.storage">
    <strong>Persistent storage</strong> — Configurable PVCs with storage class selection
  </span>
- <span data-i18n="docsCharts.mongodb.feature.arbiter">
    <strong>Arbiter support</strong> — Optional arbiter for replica set elections
  </span>

## <span data-i18n="docsCharts.heading.installation">Installation</span>

**<span data-i18n="docsCharts.install.httpsRepo">HTTPS repository:</span>**

```bash
helm repo add helmforge https://repo.helmforge.dev
helm repo update
helm install my-mongo helmforge/mongodb
```

**<span data-i18n="docsCharts.install.ociRegistry">OCI registry:</span>**

```bash
helm install my-mongo oci://ghcr.io/helmforgedev/helm/mongodb
```

## <span data-i18n="docsCharts.heading.deploymentExamples">Deployment Examples</span>

<DocsTabs tabs={[
  { id: 'standalone', label: 'Standalone' },
  { id: 'replicaset', label: 'Replica Set' },
  { id: 'arbiter', label: 'With Arbiter' },

<div data-tab-panel="standalone">

```yaml
# values.yaml
architecture: standalone

auth:
  rootPassword: 'my-secret-password'
  database: myapp
  username: myuser
  password: 'user-password'

persistence:
  size: 20Gi

metrics:
  enabled: true
  serviceMonitor:
    enabled: true
```

</div>

<div data-tab-panel="replicaset">

```yaml
# values.yaml
architecture: replicaset

auth:
  rootPassword: 'my-secret-password'
  database: myapp
  username: myuser
  password: 'user-password'

replicaCount: 3

persistence:
  size: 20Gi
```

</div>

<div data-tab-panel="arbiter">

```yaml
# values.yaml
architecture: replicaset

auth:
  rootPassword: 'my-secret-password'
  database: myapp

replicaCount: 2

persistence:
  size: 20Gi

arbiter:
  enabled: true
```

<Callout type="info" title="When to use an arbiter">
  <span data-i18n="docsCharts.mongodb.arbiterCallout">
    An arbiter participates in elections but holds no data. Use it when you have an even number of data-bearing members
    to ensure a majority for elections. With 3 data members, an arbiter is not needed.
  </span>
</Callout>

</div>

</DocsTabs>

## <span data-i18n="docsCharts.heading.upgradeNotes">Upgrade Notes</span>

<Callout type="warning" title="Replica set reconfiguration">
  <span data-i18n="docsCharts.mongodb.upgrade.callout">
    Changing `replicaCount` triggers a replica set reconfiguration. MongoDB handles this gracefully, but scaling down
    removes members — ensure no data is exclusively on the removed members.
  </span>
</Callout>

- <span data-i18n="docsCharts.mongodb.upgrade.note1">
    Upgrading from standalone to replicaset requires re-initialization (dump and restore)
  </span>
- <span data-i18n="docsCharts.mongodb.upgrade.note2">
    Auth credentials are set during first initialization and cannot be changed via values on upgrade
  </span>
- <span data-i18n="docsCharts.mongodb.upgrade.note3">
    WiredTiger cache size defaults to 50% of available memory — set `resources.limits.memory` accordingly
  </span>

## <span data-i18n="docsCharts.heading.commonIssues">Common Issues</span>

<Callout type="danger" title="Replica set members stuck in RECOVERING">
  <span data-i18n="docsCharts.mongodb.issue.recovering">
    This usually means initial sync is still in progress for large datasets. Check sync progress with `rs.status()`
    inside the mongo shell. If it persists, check disk IOPS and network bandwidth.
  </span>
</Callout>

<Callout type="tip" title="Connection string for replica sets">
  <span data-i18n="docsCharts.mongodb.issue.connectionString.before">
    Use the full replica set connection string with all members listed, or use the headless service DNS:
  </span>{' '}
  <code>mongodb://user:pass@my-mongo-mongodb-headless.namespace.svc:27017/mydb?replicaSet=rs0</code>
</Callout>
