# Homarr

Helm chart for deploying [Homarr](https://homarr.dev/) modern application dashboard on Kubernetes using the official [`ghcr.io/homarr-labs/homarr`](https://github.com/homarr-labs/homarr/pkgs/container/homarr) container image.

## <span data-i18n="docsCharts.heading.keyFeatures">Key Features</span>

- <span data-i18n="docsCharts.auto.homarr.b0">Official Homarr image from `ghcr.io/homarr-labs/homarr`</span>
- <span data-i18n="docsCharts.common.feature.databaseBackendsAutoDetection">
    **Database backends** SQLite3 (default), PostgreSQL, or MySQL with auto-detection
  </span>
- <span data-i18n="docsCharts.common.feature.postgresqlMysqlSubchartsOptional">
    **PostgreSQL and MySQL subcharts** optional bundled database deployments
  </span>
- <span data-i18n="docsCharts.auto.homarr.b1">
    Encryption key management auto-generated or existing secret for `SECRET_ENCRYPTION_KEY`
  </span>
- <span data-i18n="docsCharts.auto.homarr.b2">
    Kubernetes integration optional workload discovery via `ENABLE_KUBERNETES`
  </span>
- <span data-i18n="docsCharts.auto.homarr.b3">External Redis optional external Redis for multi-instance setups</span>
- <span data-i18n="docsCharts.common.feature.s3CompatibleBackupDatabaseAware">
    **S3-compatible backup** database-aware CronJob (SQLite tar, pg_dump, mysqldump)
  </span>
- <span data-i18n="docsCharts.common.feature.ingressConfigurableTls">
    **Ingress support** configurable ingress with TLS
  </span>

## <span data-i18n="docsCharts.heading.installation">Installation</span>

**HTTPS Repository**

```bash
helm repo add helmforge https://repo.helmforge.dev
helm repo update
helm install homarr helmforge/homarr
```

**OCI Registry**

```bash
helm install homarr oci://ghcr.io/helmforgedev/helm/homarr
```

## <span data-i18n="docsCharts.heading.basicExample">Basic Example</span>

```yaml
homarr:
enableKubernetes: true
postgresql:
enabled: true
auth:
  database: homarr
username: homarr
password: 'db-password'
ingress:
enabled: true
ingressClassName: traefik
hosts:
  - host: dash.example.com
paths:
  - path: /
pathType: Prefix
tls:
  - secretName: homarr-tls
hosts:
  - dash.example.com
backup:
enabled: true
schedule: '0 3 * * *'
s3:
  endpoint:
https://s3.amazonaws.com
bucket: my-backups
prefix: homarr
existingSecret: homarr-s3-credentials
```

## <span data-i18n="docsCharts.heading.databaseAutoDetection">Database Auto-Detection</span>

When `database.mode` is `auto` (default), the chart detects which database to use:1. If `database.external.host` is set → **external** database2. If `postgresql.enabled` is `true` → **PostgreSQL subchart**3. If `mysql.enabled` is `true` → **MySQL subchart**4. Otherwise → **SQLite3** (zero configuration)

## <span data-i18n="docsCharts.heading.encryptionKey">Encryption Key</span>

Homarr requires a `SECRET_ENCRYPTION_KEY` for encrypting integration secrets. The chart auto-generates a 64-character hex key on first install if not provided. To set your own:

```bash
openssl rand -hex 32
```

Then pass it via `encryption.key` or an existing secret.
