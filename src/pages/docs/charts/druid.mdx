# Apache Druid

<span data-i18n="docsCharts.druid.intro.before">Deploy</span> [Apache Druid](https://druid.apache.org/)
<span data-i18n="docsCharts.druid.intro.after">
  on Kubernetes — a high-performance, real-time analytics database designed for fast slice-and-dice analytics on large
  datasets, commonly used for BI dashboards and real-time data exploration.
</span>

## <span data-i18n="docsCharts.heading.keyFeatures">Key Features</span>

- <span data-i18n="docsCharts.druid.feature.sixComponentArchitecture">
    6-component architecture — coordinator, overlord, broker, router, historical, middlemanager
  </span>
- <span data-i18n="docsCharts.druid.feature.routerConsole">
    Router web console — built-in Druid console accessible via ingress
  </span>
- <span data-i18n="docsCharts.druid.feature.postgresqlMetadataStore">
    PostgreSQL metadata store — bundled subchart or external PostgreSQL/MySQL
  </span>
- <span data-i18n="docsCharts.druid.feature.zookeeperCoordination">
    ZooKeeper coordination — bundled subchart or external ZooKeeper
  </span>
- <span data-i18n="docsCharts.druid.feature.deepStorage">Deep storage — local or S3-compatible (MinIO, AWS S3)</span>
- <span data-i18n="docsCharts.druid.feature.persistentVolumes">
    Persistent volumes — segment cache (historical) and task storage (middlemanager)
  </span>
- <span data-i18n="docsCharts.druid.feature.perComponentScaling">
    Per-component scaling — independent replica counts and JVM tuning
  </span>
- <span data-i18n="docsCharts.druid.feature.healthProbes">
    Health probes — liveness on /status/health, readiness on /status/selfDiscovered
  </span>
- <span data-i18n="docsCharts.druid.feature.ingressSupport">
    Ingress support — configurable class (traefik, nginx, etc.)
  </span>

## <span data-i18n="docsCharts.heading.installation">Installation</span>

**<span data-i18n="docsCharts.install.httpsRepo">HTTPS repository:</span>**

```bash
helm repo add helmforge https://repo.helmforge.dev
helm repo update
helm install druid helmforge/druid -f values.yaml
```

**<span data-i18n="docsCharts.install.ociRegistry">OCI registry:</span>**

```bash
helm install druid oci://ghcr.io/helmforgedev/helm/druid -f values.yaml
```

## <span data-i18n="docsCharts.heading.basicExample">Basic Example</span>

```yaml
coordinator:
  enabled: true

broker:
  enabled: true

router:
  enabled: true

historical:
  enabled: true
  persistence:
    size: 20Gi

ingress:
  enabled: true
  ingressClassName: traefik
  hosts:
    - host: druid.example.com
      paths:
        - path: /
          pathType: Prefix
```

## <span data-i18n="docsCharts.heading.s3DeepStorage">S3 Deep Storage</span>

## <span data-i18n="docsCharts.heading.externalServices">External Services</span>

```yaml
metadata:
  mode: external
  external:
    type: postgresql
    host: postgres.example.internal
    port: 5432
    name: druid
    username: druid
    existingSecret: druid-db

zookeeperConfig:
  mode: external
  external:
    hosts: 'zk1.example.internal:2181,zk2.example.internal:2181'

postgresql:
  enabled: false

zookeeper:
  enabled: false
```

## <span data-i18n="docsCharts.heading.operationalNotes">Operational Notes</span>

- <span data-i18n="docsCharts.druid.note.memoryIntensive">
    Druid is memory-intensive — all 6 components together require 2GB+ RAM minimum
  </span>
- <span data-i18n="docsCharts.druid.note.tuneJvm">
    Tune JVM heap per component via `javaOpts` for production sizing
  </span>
- <span data-i18n="docsCharts.druid.note.routerIngressTarget">
    The router component serves the web console and is the default ingress target
  </span>
- <span data-i18n="docsCharts.druid.note.s3ForProduction">
    Use S3 deep storage for production; local deep storage is only suitable for development
  </span>
- <span data-i18n="docsCharts.druid.note.historicalCache">
    Historical nodes cache segments on disk — size the PVC based on data volume
  </span>
