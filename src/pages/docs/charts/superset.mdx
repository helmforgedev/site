# Apache Superset

<span data-i18n="docsCharts.superset.intro.before">Deploy</span> [Apache Superset](https://superset.apache.org/)
<span data-i18n="docsCharts.superset.intro.after">
  on Kubernetes — a modern, enterprise-ready business intelligence platform for data exploration and visualization with
  60+ database connectors.
</span>

## <span data-i18n="docsCharts.heading.keyFeatures">Key Features</span>

- <span data-i18n="docsCharts.superset.feature.threeComponent">
    Three-component architecture — Gunicorn web, Celery workers, Celery beat scheduler for independent scaling
  </span>
- <span data-i18n="docsCharts.superset.feature.initJob">
    Init Job — automatic database migration and admin user creation
  </span>
- <span data-i18n="docsCharts.superset.feature.postgresqlMetadata">
    PostgreSQL metadata store — bundled subchart or external connection
  </span>
- <span data-i18n="docsCharts.superset.feature.redisBroker">
    Redis broker — bundled subchart for Celery task queue or external
  </span>
- <span data-i18n="docsCharts.superset.feature.configurable">
    Configurable — superset_config.py delivered via ConfigMap
  </span>
- <span data-i18n="docsCharts.superset.feature.autoSecrets">
    Auto-generated secrets — admin password, Flask SECRET_KEY, database and Redis passwords
  </span>
- <span data-i18n="docsCharts.superset.feature.healthProbes">
    Health probes — startup, liveness, and readiness on /health
  </span>
- <span data-i18n="docsCharts.superset.feature.ingressSupport">
    Ingress support — configurable class (traefik, nginx, etc.)
  </span>

## <span data-i18n="docsCharts.heading.installation">Installation</span>

**<span data-i18n="docsCharts.install.httpsRepo">HTTPS repository:</span>**

```bash
helm repo add helmforge https://repo.helmforge.dev
helm repo update
helm install superset helmforge/superset -f values.yaml
```

**<span data-i18n="docsCharts.install.ociRegistry">OCI registry:</span>**

```bash
helm install superset oci://ghcr.io/helmforgedev/helm/superset -f values.yaml
```

## <span data-i18n="docsCharts.heading.basicExample">Basic Example</span>

```yaml
superset:
  adminUsername: admin
  adminPassword: 'my-secure-password'
  secretKey: 'my-flask-secret-key'

ingress:
  enabled: true
  ingressClassName: traefik
  hosts:
    - host: superset.example.com
      paths:
        - path: /
          pathType: Prefix
```

## <span data-i18n="docsCharts.heading.scaledWorkers">Scaled Workers</span>

```yaml
web:
  replicaCount: 2
  workers: 4

worker:
  replicaCount: 3
  concurrency: 4

beat:
  enabled: true
```

## <span data-i18n="docsCharts.heading.externalDatabase">External Database</span> and Redis

```yaml
database:
  mode: external
  external:
    type: postgresql
    host: postgres.example.internal
    port: 5432
    name: superset
    username: superset
    existingSecret: superset-db

redisConfig:
  mode: external
  external:
    host: redis.example.internal
    port: 6379

postgresql:
  enabled: false

redis:
  enabled: false
```

## <span data-i18n="docsCharts.heading.operationalNotes">Operational Notes</span>

- <span data-i18n="docsCharts.superset.note.initJobRuns">
    The init job runs database migrations and creates the admin user on every install/upgrade
  </span>
- <span data-i18n="docsCharts.superset.note.independentScaling">
    Web, worker, and beat components scale independently
  </span>
- <span data-i18n="docsCharts.superset.note.extraConfig">
    Configure `superset.extraConfig` for custom superset_config.py settings
  </span>
- <span data-i18n="docsCharts.superset.note.loadExamples">
    Use `superset.loadExamples: true` for demonstration environments
  </span>
