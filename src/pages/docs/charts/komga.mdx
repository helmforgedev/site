# Komga

Deploy Komga on Kubernetes using the official [gotson/komga](https://hub.docker.com/r/gotson/komga) Docker image. A media server for comics, mangas, BDs, magazines, and eBooks with OPDS support and a modern web reader.

## <span data-i18n="docsCharts.heading.keyFeatures">Key Features</span>

- <span data-i18n="docsCharts.auto.komga.b0">SQLite Database — zero database configuration, persistent via PVC</span>
- <span data-i18n="docsCharts.auto.komga.b1">Dual Persistent Volumes — separate PVCs for config and library data</span>
- <span data-i18n="docsCharts.auto.komga.b2">Java Memory Tuning — configurable JVM heap via `JAVA_TOOL_OPTIONS`</span>
- <span data-i18n="docsCharts.auto.komga.b3">
    <span data-i18n="docsCharts.common.timezone">Timezone</span> Support — configurable via `TZ` environment variable
  </span>
- <span data-i18n="docsCharts.auto.komga.b4">Scheduled Backups — config/database tar archive with S3 upload</span>
- <span data-i18n="docsCharts.common.feature.ingressTlsCertManager">**Ingress Support** — TLS with cert-manager</span>

## <span data-i18n="docsCharts.heading.installation">Installation</span>

**<span data-i18n="docsCharts.install.httpsRepo">HTTPS repository:</span>**

```bash
helm repo add helmforge https://repo.helmforge.dev
helm repo update
helm install komga helmforge/komga
```

**<span data-i18n="docsCharts.install.ociRegistry">OCI registry:</span>**

```bash
helm install komga oci://ghcr.io/helmforgedev/helm/komga
```

## <span data-i18n="docsCharts.heading.basicExample">Basic Example</span>

```yaml
# values.yaml
persistence:
config:
  enabled: true
size: 2Gi
data:
  enabled: true
size: 100Gi
```

## <span data-i18n="docsCharts.heading.productionExample">Production Example</span>

```yaml
komga:
timezone: 'America/Sao_Paulo'
javaMemory: '-Xmx2g'
sessionTimeout: '7d'
persistence:
config:
  size: 5Gi
data:
  size:
500Gi
ingress:
  enabled: true
ingressClassName: traefik
hosts:
  - host: komga.example.com
paths:
  - path: /
pathType: Prefix
tls:
  - secretName: komga-tls
hosts:
  - komga.example.com
backup:
enabled: true
schedule: '0 3 * * *'
s3:
  endpoint:
https://s3.amazonaws.com
bucket: my-backups
prefix: komga
existingSecret: komga-s3-credentials
```
