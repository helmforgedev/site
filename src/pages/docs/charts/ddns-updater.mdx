# DDNS Updater

Deploy [ddns-updater](https://github.com/qdm12/ddns-updater) on Kubernetes using the official [qmcgaw/ddns-updater](https://hub.docker.com/r/qmcgaw/ddns-updater) Docker image. Keep DNS A/AAAA records updated across 50+ providers with a responsive web UI.

## <span data-i18n="docsCharts.heading.keyFeatures">Key Features</span>

- <span data-i18n="docsCharts.auto.ddns_updater.b0">
    50+ DNS providers — Cloudflare, Route53, DuckDNS, Namecheap, GoDaddy, Hetzner, and more
  </span>
- <span data-i18n="docsCharts.auto.ddns_updater.b1">Web UI — responsive dashboard for monitoring update status</span>
- <span data-i18n="docsCharts.auto.ddns_updater.b2">
    Multi-provider — manage records across different providers in a single deployment
  </span>
- <span data-i18n="docsCharts.auto.ddns_updater.b3">Persistent history — update history stored in a PVC</span>
- <span data-i18n="docsCharts.auto.ddns_updater.b4">Existing secrets — bring your own Secret for credentials</span>
- <span data-i18n="docsCharts.auto.ddns_updater.b5">Ingress support — expose the web UI with TLS</span>

## <span data-i18n="docsCharts.heading.installation">Installation</span>

**<span data-i18n="docsCharts.install.httpsRepo">HTTPS repository:</span>**

```bash
helm repo add helmforge https://repo.helmforge.dev
helm repo update
helm install ddns-updater helmforge/ddns-updater -f values.yaml
```

**<span data-i18n="docsCharts.install.ociRegistry">OCI registry:</span>**

```bash
helm install ddns-updater oci://ghcr.io/helmforgedev/helm/ddns-updater -f values.yaml
```

## <span data-i18n="docsCharts.heading.basicExample">Basic Example</span> (Cloudflare)

```yaml
# values.yaml
config:
settings:
  - provider: cloudflare
zone_identifier: 'your-zone-id'
domain: 'example.com'
host: '@'
ttl: 300
token: 'your-cloudflare-api-token'
proxied: false
ip_version: 'ipv4'
```

Access the web UI with `kubectl port-forward svc/<release>-ddns-updater 8000:80`.

## <span data-i18n="docsCharts.heading.multiProviderExample">Multi-Provider Example</span>

```yaml
config:
settings:
  - provider: cloudflare
zone_identifier: 'cf-zone-id'
domain: 'example.com'
host: '@'
token: 'cf-token'
proxied: true
  - provider: duckdns
domain: 'myhost.duckdns.org'
token: 'duck-token'
  - provider: namecheap
domain: 'example.org'
host: 'home'
password: 'ddns-password'
```
