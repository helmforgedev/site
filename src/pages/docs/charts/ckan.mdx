# CKAN

Deploy [CKAN](https://ckan.org/) on Kubernetes — the world's leading open-source data management system for powering data hubs and data portals with full catalog and API capabilities.

## <span data-i18n="docsCharts.heading.keyFeatures">Key Features</span>
- <span data-i18n="docsCharts.auto.ckan.b0">CKAN application — uWSGI-based Python application on port 5000</span>
- <span data-i18n="docsCharts.auto.ckan.b1">DataPusher — optional automatic resource loading for CSV, Excel to DataStore</span>
- <span data-i18n="docsCharts.auto.ckan.b2">Built-in Solr — StatefulSet with CKAN-specific schema configuration</span>
- <span data-i18n="docsCharts.auto.ckan.b3">PostgreSQL metadata store — bundled subchart or external connection</span>
- <span data-i18n="docsCharts.auto.ckan.b4">Redis task queue — bundled subchart or external connection</span>
- <span data-i18n="docsCharts.auto.ckan.b5">Auto-generated secrets — sysadmin password, session secret, JWT secret</span>
- <span data-i18n="docsCharts.auto.ckan.b6">Persistent storage — uploaded datasets at /var/lib/ckan</span>
- <span data-i18n="docsCharts.auto.ckan.b7">Plugin system — configurable CKAN plugins list</span>
- <span data-i18n="docsCharts.common.feature.ingressCertManager">**Ingress support** — TLS via cert-manager with configurable ingress class</span>

## <span data-i18n="docsCharts.heading.installation">Installation</span>

**<span data-i18n="docsCharts.install.httpsRepo">HTTPS repository:</span>**

```bash
helm repo add helmforge https://repo.helmforge.dev
helm repo update
helm install ckan helmforge/ckan -f values.yaml
```

**<span data-i18n="docsCharts.install.ociRegistry">OCI registry:</span>**

```bash
helm install ckan oci://ghcr.io/helmforgedev/helm/ckan -f values.yaml
```

## <span data-i18n="docsCharts.heading.basicExample">Basic Example</span>

```yaml
ckan:
siteUrl:
  https://data.example.com
sysadminName: admin
sysadminPassword: 'my-secure-password'
ingress:
enabled: true
ingressClassName: traefik
hosts:
  - host: data.example.com
paths:
  - path: /
pathType: Prefix
```

## <span data-i18n="docsCharts.heading.externalServices">External Services</span>

```yaml
database:
mode: external
external:
  type: postgresql
host: postgres.example.internal
port: 5432
name: ckan
username: ckan
existingSecret:
  ckan-dbredisConfig:
mode: external
external:
  host: redis.example.internal
port: 6379
postgresql:
enabled: false
redis:
enabled: false
```

## <span data-i18n="docsCharts.heading.operationalNotes">Operational Notes</span>
- Solr is deployed as a StatefulSet with CKAN-specific schema — do not share with other applications- DataPusher is optional but recommended for automatic resource indexing- health probes target /api/action/status_show- set `ckan.siteUrl` to the actual public URL for correct link generation
