# Generic

A multi-purpose Helm chart for deploying any container workload on Kubernetes without writing boilerplate
templates. One chart for Deployments, StatefulSets, DaemonSets, Jobs, CronJobs, multi-container pods,
PVC management, ConfigMaps, and arbitrary extra manifests with Helm `tpl` support.

<Callout type="info" title="Jobs and CronJobs are separate lists — not workload types">
  Unlike some charts, `workload.type` only accepts `Deployment`, `StatefulSet`, and `DaemonSet`. One-shot Jobs and
  scheduled CronJobs are configured via the separate `jobs[]` and `cronjobs[]` arrays. Set `workload.enabled: false` to
  deploy a release that contains only Jobs or CronJobs with no long-running workload.
</Callout>

## Key Features

- **All workload types** — Deployment, StatefulSet, DaemonSet, Job, CronJob
- **Multi-container pods** — `containers[]` array with per-container image, env, ports, and probes
- **Flexible persistence** — separate `volumes`, `storage` (PVCs), and `mounts` arrays
- **Inline ConfigMaps** — create ConfigMaps directly from `values.yaml`
- **Global and per-container env** — `env`/`envFrom` at root apply to all containers
- **HPA and VPA** — horizontal and vertical pod autoscaling
- **PDB** — PodDisruptionBudget for production availability
- **Helm-templated extraManifests** — inject any Kubernetes manifest with `tpl` support
- **ServiceMonitor** — Prometheus Operator integration

## Installation

**HTTPS repository:**

```bash
helm repo add helmforge https://repo.helmforge.dev
helm repo update
helm install my-app helmforge/generic -f values.yaml
```

**OCI registry:**

```bash
helm install my-app oci://ghcr.io/helmforgedev/helm/generic -f values.yaml
```

workload:
  enabled: true
  type: StatefulSet
  podManagementPolicy: OrderedReady

workload:

<div data-tab-panel="job-cronjob">

```yaml
# values.yaml — Release with only Jobs and CronJobs (no long-running workload)
workload:
  enabled: false # no Deployment/StatefulSet/DaemonSet

# One-shot database migration Job
jobs:
  - name: db-migrate
    image:
      repository: docker.io/my-org/my-app
      tag: '1.0'
    command: ['npm', 'run', 'migrate']
    backoffLimit: 3
    activeDeadlineSeconds: 600

# Scheduled cleanup CronJob
cronjobs:
  - name: cleanup
    schedule: '0 2 * * *'
    timeZone: America/Sao_Paulo
    image:
      repository: docker.io/my-org/my-app
      tag: '1.0'
    command: ['npm', 'run', 'cleanup']
    concurrencyPolicy: Forbid
    suspend: false

# Global env applied to all Jobs and CronJobs
env:
  - name: DATABASE_URL
    valueFrom:
      secretKeyRef:
        name: app-db-secret
        key: database-url
```

</div>

workload:

  enabled: true

## Configuration Reference

### Core

| Parameter          | Type    | Default | Description                                          |
| ------------------ | ------- | ------- | ---------------------------------------------------- |
| `replicaCount`     | integer | `1`     | Number of pod replicas. Ignored when HPA is enabled. |
| `nameOverride`     | string  | `""`    | Override the chart name.                             |
| `fullnameOverride` | string  | `""`    | Override the full release name.                      |
| `commonLabels`     | object  | `{}`    | Extra labels added to all resources.                 |

### Workload

| Parameter                       | Type    | Default      | Description                                                             |
| ------------------------------- | ------- | ------------ | ----------------------------------------------------------------------- |
| `workload.enabled`              | boolean | `true`       | Deploy the long-running workload. Set to `false` for Job-only releases. |
| `workload.type`                 | string  | `Deployment` | Workload kind: `Deployment`, `StatefulSet`, or `DaemonSet`.             |
| `workload.podManagementPolicy`  | string  | —            | StatefulSet pod management policy (`OrderedReady` or `Parallel`).       |
| `workload.volumeClaimTemplates` | array   | —            | StatefulSet per-pod PVC templates.                                      |
| `workload.updateStrategy`       | object  | —            | Update strategy for StatefulSet or DaemonSet.                           |

### Image

| Parameter          | Type   | Default                               | Description                          |
| ------------------ | ------ | ------------------------------------- | ------------------------------------ |
| `image.repository` | string | `container.registry.io/project/image` | Default container image.             |
| `image.tag`        | string | `latest`                              | Image tag.                           |
| `image.pullPolicy` | string | `Always`                              | Image pull policy.                   |
| `imagePullSecrets` | array  | `[]`                                  | Pull secrets for private registries. |

### Containers

Each entry in `containers[]` becomes a container in the pod spec:

| Field             | Description                                                            |
| ----------------- | ---------------------------------------------------------------------- |
| `name`            | Container name.                                                        |
| `image`           | Per-container image override. Use `{}` to inherit the top-level image. |
| `ports`           | List of `containerPort` entries.                                       |
| `env`             | Per-container environment variables.                                   |
| `envFrom`         | Per-container `configMapRef` or `secretRef` sources.                   |
| `command`         | Container command override.                                            |
| `args`            | Container args override.                                               |
| `livenessProbe`   | Per-container liveness probe.                                          |
| `readinessProbe`  | Per-container readiness probe.                                         |
| `startupProbe`    | Per-container startup probe.                                           |
| `resources`       | Per-container resource requests and limits.                            |
| `securityContext` | Per-container security context.                                        |
| `volumeMounts`    | Per-container volume mounts.                                           |
| `lifecycle`       | Per-container lifecycle hooks.                                         |

`initContainers[]` accepts the same fields and runs before the main containers start.

### Global Environment

Applied to **all containers**, including those defined in `jobs[]` and `cronjobs[]`:

| Parameter | Type  | Default | Description                                                       |
| --------- | ----- | ------- | ----------------------------------------------------------------- |
| `env`     | array | `[]`    | Global environment variables injected into all containers.        |
| `envFrom` | array | `[]`    | Global `configMapRef` and `secretRef` sources for all containers. |

### Persistence

The generic chart separates persistence into three independent lists:

| Parameter               | Description                                                              |
| ----------------------- | ------------------------------------------------------------------------ |
| `persistence.volumes[]` | Inline volume definitions (emptyDir, configMap, secret, hostPath, etc.). |
| `persistence.storage[]` | PVCs created by the chart (specify `name`, `capacity`, `accessMode`).    |
| `persistence.mounts[]`  | Volume mount definitions mapping volume names to container paths.        |

Example:

```yaml
persistence:
  storage:
    - name: data
      capacity: 10Gi
      accessMode: ReadWriteOnce
      storageClassName: standard

  volumes:
    - name: config
      configMap:
        name: my-app-config

  mounts:
    - name: data
      mountPath: /app/data
    - name: config
      mountPath: /etc/config
      readOnly: true
```

### ConfigMaps

| Parameter           | Description                                     |
| ------------------- | ----------------------------------------------- |
| `configMaps[]`      | Inline ConfigMaps created by the chart.         |
| `configMaps[].name` | ConfigMap name.                                 |
| `configMaps[].data` | Key-value pairs for the ConfigMap data section. |

### Service

| Parameter             | Type    | Default     | Description                             |
| --------------------- | ------- | ----------- | --------------------------------------- |
| `service.type`        | string  | `ClusterIP` | Service type.                           |
| `service.port`        | integer | `8080`      | Service port.                           |
| `service.targetPort`  | integer | `8080`      | Container port the Service forwards to. |
| `service.extraPorts`  | array   | `[]`        | Additional ports (e.g. metrics, admin). |
| `service.annotations` | object  | `{}`        | Annotations for the Service.            |

### Ingress

| Parameter                  | Type    | Default   | Description                  |
| -------------------------- | ------- | --------- | ---------------------------- |
| `ingress.enabled`          | boolean | `false`   | Enable an Ingress resource.  |
| `ingress.ingressClassName` | string  | `traefik` | Ingress class name.          |
| `ingress.annotations`      | object  | `{}`      | Annotations for the Ingress. |
| `ingress.hosts`            | array   | `[]`      | Host and path rules.         |
| `ingress.tls`              | array   | `[]`      | TLS configuration.           |

### Update Strategy

| Parameter                                     | Type   | Default         | Description                                 |
| --------------------------------------------- | ------ | --------------- | ------------------------------------------- |
| `updateStrategy.type`                         | string | `RollingUpdate` | Deployment update strategy type.            |
| `updateStrategy.rollingUpdate.maxSurge`       | string | `25%`           | Max pods above desired count during update. |
| `updateStrategy.rollingUpdate.maxUnavailable` | string | `25%`           | Max unavailable pods during update.         |

### Autoscaling

| Parameter         | Type    | Default | Description                                     |
| ----------------- | ------- | ------- | ----------------------------------------------- |
| `hpa.enabled`     | boolean | `false` | Enable Horizontal Pod Autoscaler.               |
| `hpa.minReplicas` | integer | —       | Minimum replica count.                          |
| `hpa.maxReplicas` | integer | —       | Maximum replica count.                          |
| `hpa.metrics`     | array   | —       | HPA metric targets (CPU/memory utilization).    |
| `vpa.enabled`     | boolean | `false` | Enable Vertical Pod Autoscaler.                 |
| `vpa.updateMode`  | string  | `Off`   | VPA update mode (`Off` = recommendations only). |

### PodDisruptionBudget

| Parameter            | Type    | Default | Description                                  |
| -------------------- | ------- | ------- | -------------------------------------------- |
| `pdb.enabled`        | boolean | `false` | Create a PodDisruptionBudget.                |
| `pdb.minAvailable`   | integer | —       | Minimum available pods during disruptions.   |
| `pdb.maxUnavailable` | integer | —       | Maximum unavailable pods during disruptions. |

### ServiceMonitor

| Parameter                    | Description                                           |
| ---------------------------- | ----------------------------------------------------- |
| `serviceMonitor.enabled`     | Create a Prometheus Operator ServiceMonitor.          |
| `serviceMonitor.labels`      | Extra labels for the ServiceMonitor.                  |
| `serviceMonitor.endpoints[]` | Scrape endpoints with `port`, `path`, and `interval`. |

### Jobs

Each entry in `jobs[]` creates a Kubernetes `Job`:

| Field                   | Description                                         |
| ----------------------- | --------------------------------------------------- |
| `name`                  | Job name.                                           |
| `image`                 | Job container image.                                |
| `command`               | Command to run.                                     |
| `args`                  | Arguments.                                          |
| `backoffLimit`          | Number of retries before marking the Job as failed. |
| `activeDeadlineSeconds` | Maximum duration before the Job is terminated.      |
| `env`                   | Per-Job environment variables (merged with global). |
| `envFrom`               | Per-Job envFrom sources.                            |

### CronJobs

Each entry in `cronjobs[]` creates a Kubernetes `CronJob`:

| Field               | Description                                                |
| ------------------- | ---------------------------------------------------------- |
| `name`              | CronJob name.                                              |
| `schedule`          | Cron expression.                                           |
| `timeZone`          | IANA timezone for the schedule (e.g. `America/Sao_Paulo`). |
| `image`             | Container image.                                           |
| `command`           | Command to run.                                            |
| `concurrencyPolicy` | `Forbid`, `Replace`, or `Allow`.                           |
| `suspend`           | Suspend the CronJob without deleting it.                   |

### Extra Manifests

<Callout type="tip" title="extraManifests supports Helm tpl templating">
  Unlike most charts, `extraManifests` in the Generic chart processes each manifest through the Helm
  `tpl` function, enabling the use of template expressions like
  `{{ include "chart.fullname" . }}-extra` inside the manifest definitions.
</Callout>

| Parameter        | Type  | Default | Description                                                                                   |
| ---------------- | ----- | ------- | --------------------------------------------------------------------------------------------- |
| `extraManifests` | array | `[]`    | Arbitrary Kubernetes manifests rendered via Helm `tpl`. Supports all Helm template functions. |

### Resources and Security

| Parameter            | Type   | Default | Description                                                    |
| -------------------- | ------ | ------- | -------------------------------------------------------------- |
| `resources`          | object | `{}`    | Default CPU and memory requests and limits for all containers. |
| `podSecurityContext` | object | `{}`    | Pod-level security context.                                    |
| `securityContext`    | object | `{}`    | Default container-level security context.                      |

### Scheduling

| Parameter                       | Type    | Default | Description                            |
| ------------------------------- | ------- | ------- | -------------------------------------- |
| `nodeSelector`                  | object  | `{}`    | Node selector for scheduling.          |
| `tolerations`                   | array   | `[]`    | Tolerations for scheduling.            |
| `affinity`                      | object  | `{}`    | Affinity rules.                        |
| `topologySpreadConstraints`     | array   | `[]`    | Topology spread constraints.           |
| `priorityClassName`             | string  | `""`    | PriorityClass for the pod.             |
| `terminationGracePeriodSeconds` | integer | `30`    | Termination grace period.              |
| `podLabels`                     | object  | `{}`    | Extra labels for the pod.              |
| `podAnnotations`                | object  | `{}`    | Extra annotations for the pod.         |
| `annotations`                   | object  | `{}`    | Annotations for the workload resource. |
