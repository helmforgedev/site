# GenericA multi-purpose Helm chart that can deploy virtually any container workload on Kubernetes. Instead of writing boilerplate templates for each application, the Generic chart lets you configure everything through `values.yaml`.

## <span data-i18n="docsCharts.heading.keyFeatures">Key Features</span>
- <span data-i18n="docsCharts.auto.generic.b0">Multiple workload types — Deployment, StatefulSet, DaemonSet, Job, CronJob</span>
- <span data-i18n="docsCharts.auto.generic.b1">Flexible container configuration — Ports, environment variables, volumes, probes</span>
- <span data-i18n="docsCharts.auto.generic.b2">Ingress support — Configurable with `ingressClassName` and TLS</span>
- <span data-i18n="docsCharts.auto.generic.b3">Service accounts — Create or reference existing service accounts</span>
- <span data-i18n="docsCharts.auto.generic.b4">Init containers and sidecars — Full support for multi-container pods</span>
- <span data-i18n="docsCharts.auto.generic.b5">HPA and PDB — Autoscaling and disruption budgets</span>

## <span data-i18n="docsCharts.heading.installation">Installation</span>

**<span data-i18n="docsCharts.install.httpsRepo">HTTPS repository:</span>**

```bash
helm repo add helmforge https://repo.helmforge.dev
helm repo update
helm install my-app helmforge/generic -f values.yaml
```

**<span data-i18n="docsCharts.install.ociRegistry">OCI registry:</span>**

```bash
helm install my-app oci://ghcr.io/helmforgedev/helm/generic -f values.yaml
```

## <span data-i18n="docsCharts.heading.basicExample">Basic Example</span>

Deploy a simple web application:

```yaml
# values.yaml
image:
repository: nginx
tag: '1.27'
service:
port: 80
ingress:
enabled: true
ingressClassName: traefik
hosts:
  - host: app.example.com
paths:
  - path: /
pathType: Prefix
resources:
requests:
  cpu: 100m
memory: 128Mi
limits:
  memory: 256Mi
```
