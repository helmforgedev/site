# MariaDB

<span data-i18n="docsCharts.mariadb.intro.before">Helm chart for deploying</span> [MariaDB](https://mariadb.org/)
<span data-i18n="docsCharts.mariadb.intro.after">
  on Kubernetes using the official `mariadb` Docker image. Supports standalone and GTID-based replication architectures.
</span>

## <span data-i18n="docsCharts.heading.keyFeatures">Key Features</span>

- <span data-i18n="docsCharts.mariadb.feature.officialImage">Official MariaDB image from Docker Hub (11.4 LTS)</span>
- <span data-i18n="docsCharts.mariadb.feature.standaloneReplication">
    Standalone and replication modes with explicit configuration
  </span>
- <span data-i18n="docsCharts.mariadb.feature.gtidReplication">
    GTID-based replication using MariaDB-native `MASTER_USE_GTID=slave_pos`
  </span>
- <span data-i18n="docsCharts.mariadb.feature.configPresets">
    Configuration presets: small, medium, large, oltp, read-heavy, analytics
  </span>
- <span data-i18n="docsCharts.mariadb.feature.tls">TLS server-side encryption with optional client enforcement</span>
- <span data-i18n="docsCharts.mariadb.feature.prometheusMetrics">
    Prometheus metrics via mysqld-exporter sidecar and ServiceMonitor
  </span>
- <span data-i18n="docsCharts.mariadb.feature.s3Backup">S3-compatible backup CronJob with mariadb-dump</span>
- <span data-i18n="docsCharts.mariadb.feature.networkPolicyPdb">NetworkPolicy and PDB for production workloads</span>

## <span data-i18n="docsCharts.heading.installation">Installation</span>

**<span data-i18n="docsCharts.install.httpsRepo">HTTPS repository:</span>**

```bash
helm repo add helmforge https://repo.helmforge.dev
helm repo update
helm install mariadb helmforge/mariadb
```

**<span data-i18n="docsCharts.install.ociRegistry">OCI registry:</span>**

```bash
helm install mariadb oci://ghcr.io/helmforgedev/helm/mariadb
```

## <span data-i18n="docsCharts.heading.basicExample">Basic Example</span>

```yaml
architecture: replication

auth:
  rootPassword: 'change-me'
  database: myapp
  username: myapp
  password: 'change-me'
  replicationPassword: 'change-me'

replication:
  source:
    persistence:
      size: 50Gi
  readReplicas:
    replicaCount: 2
    persistence:
      size: 50Gi

metrics:
  enabled: true
  serviceMonitor:
    enabled: true

backup:
  enabled: true
  schedule: '0 3 * * *'
  s3:
    endpoint:
      https://s3.amazonaws.com
    bucket: my-backups
    prefix: mariadb
    existingSecret: mariadb-backup-creds
```

## <span data-i18n="docsCharts.heading.architecture">Architecture</span> Selection

<span data-i18n="docsCharts.mariadb.architectureSelection">
  When `architecture` is `standalone` (default), the chart deploys a single MariaDB instance. Set `architecture:
  replication` for one fixed source with asynchronous GTID-based read replicas.
</span>
