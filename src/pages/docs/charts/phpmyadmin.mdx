# phpMyAdmin

Web-based MySQL and MariaDB administration interface. phpMyAdmin provides a browser-based UI for managing
databases, running SQL queries, importing and exporting data, managing users and permissions, and browsing
table structures. It is stateless — no persistent storage is required.

<Callout type="danger" title="Never expose phpMyAdmin publicly without additional authentication">
  phpMyAdmin instances are actively scanned and attacked by bots on the internet. Never expose phpMyAdmin via an Ingress
  without an additional authentication layer (e.g. Traefik BasicAuth middleware, IP allowlist, or VPN). Limit access to
  internal cluster services or trusted IP ranges only.
</Callout>

## Key Features

- **MySQL and MariaDB** — connect to any accessible MySQL-compatible server
- **Single or multi-server** — connect to one fixed host or let users choose from a list
- **Stateless** — no persistent storage required, supports horizontal scaling
- **Auto-login** — optional automatic authentication using stored credentials
- **Custom configuration** — mount `config.user.inc.php` via ConfigMap for advanced settings
- **Configurable upload limit** — adjust max SQL import size
- **Ingress support** — configurable TLS via cert-manager

## Installation

**HTTPS repository:**

```bash
helm repo add helmforge https://repo.helmforge.dev
helm repo update
helm install phpmyadmin helmforge/phpmyadmin --set phpmyadmin.host=mysql.default.svc
```

**OCI registry:**

```bash
helm install phpmyadmin oci://ghcr.io/helmforgedev/helm/phpmyadmin --set phpmyadmin.host=mysql.default.svc
```

### phpMyAdmin Configuration

| Parameter                | Type    | Default | Description                                                                              |
| ------------------------ | ------- | ------- | ---------------------------------------------------------------------------------------- |
| `phpmyadmin.host`        | string  | `""`    | MySQL/MariaDB hostname to connect to (single-server mode).                               |
| `phpmyadmin.hosts`       | string  | `""`    | Comma-separated list of MySQL hostnames (multi-server mode). Leave `host` empty.         |
| `phpmyadmin.port`        | integer | `3306`  | MySQL port.                                                                              |
| `phpmyadmin.uploadLimit` | string  | `"64M"` | Maximum SQL file size for imports (e.g. `64M`, `128M`, `256M`).                          |
| `phpmyadmin.absoluteUri` | string  | `""`    | Absolute URI when behind a reverse proxy at a subpath (e.g. `https://example.com/pma/`). |
| `phpmyadmin.extraEnv`    | array   | `[]`    | Extra environment variables for advanced phpMyAdmin configuration.                       |

<Callout type="info" title="Single-server vs multi-server mode">
  Use `phpmyadmin.host` to lock phpMyAdmin to a single MySQL server — users will not be able to choose a different host.
  Use `phpmyadmin.hosts` (comma-separated) to present a server selection dropdown on the login page. When using
  multi-server mode, leave `phpmyadmin.host` empty.
</Callout>

<Callout type="warning" title="uploadLimit must match PHP and web server settings">
  The `uploadLimit` value sets the maximum SQL import file size. For large database imports, increase this value. Note
  that this setting corresponds to the PHP `upload_max_filesize` and `post_max_size` directives. Values larger than your
  proxy's body size limit (e.g. Traefik's `maxRequestBodyBytes`) will still be rejected at the ingress level.
</Callout>

### Authentication (Auto-Login)

<Callout type="danger" title="Auto-login bypasses the phpMyAdmin login form">
  Setting `auth.username` and `auth.password` (or `auth.existingSecret`) enables automatic authentication — anyone who
  accesses the phpMyAdmin URL is immediately logged into the MySQL server without entering credentials. Only use
  auto-login when phpMyAdmin is protected by an external authentication layer such as a Traefik BasicAuth middleware,
  Authelia, or VPN access restriction.
</Callout>

| Parameter                        | Type   | Default    | Description                                                         |
| -------------------------------- | ------ | ---------- | ------------------------------------------------------------------- |
| `auth.username`                  | string | `""`       | MySQL username for auto-login. Bypasses the login form if set.      |
| `auth.password`                  | string | `""`       | MySQL password for auto-login (plain text — prefer existingSecret). |
| `auth.existingSecret`            | string | `""`       | Existing Kubernetes Secret containing auto-login credentials.       |
| `auth.existingSecretUsernameKey` | string | `username` | Key in the existing secret for the MySQL username.                  |
| `auth.existingSecretPasswordKey` | string | `password` | Key in the existing secret for the MySQL password.                  |
