# AI Agent Instructions — HelmForge Site

## Overview

This is the HelmForge landing page and documentation portal. Built with Astro 6, Tailwind CSS 4, and MDX. Deployed to Cloudflare Pages at [helmforge.dev](https://helmforge.dev).

## Mandatory Rules

### Before Every Push

1. Run `npm run build` — must complete with zero errors
2. Verify new pages appear in the build output
3. Confirm sidebar navigation is updated if pages were added or removed

### Page Creation Checklist

When creating any new page:

- [ ] Add frontmatter with `layout`, `title` (unique), and `description` (50-160 chars)
- [ ] Register in `DocsLayout.astro` sidebar (`sidebarNav` or `chartNav` array)
- [ ] Verify prev/next navigation order is correct (`allPages` array)
- [ ] Run `npm run build` to confirm sitemap includes the new URL
- [ ] Sitemap is auto-generated — never edit `sitemap-*.xml` manually

When creating a new chart doc page:

- [ ] All items above, plus:
- [ ] Add to `ChartGrid.astro` on the landing page (name, slug, color, letter, description)
- [ ] Include Installation section with HTTPS repository and OCI registry subsections
- [ ] Include Key Values table
- [ ] Link to GitHub source at the end

### Page Removal Checklist

When removing a page:

- [ ] Remove the MDX file
- [ ] Remove from `DocsLayout.astro` sidebar arrays
- [ ] Remove from `ChartGrid.astro` if it was a chart
- [ ] Run `npm run build` to confirm sitemap no longer includes the URL
- [ ] Check for broken internal links from other pages

### Commit Rules

- Conventional Commits with scope `site`: `feat(site):`, `fix(site):`, `docs(site):`, `ci(site):`
- Commit subjects must be lowercase
- Never add Co-Authored-By lines to commits
- All documentation must be written in English

## Architecture

- `BaseLayout.astro` — HTML shell with meta tags, GA4, JSON-LD structured data, and global CSS import
- `DocsLayout.astro` — wraps all documentation pages with sidebar, breadcrumbs, prev/next navigation, and copy button injection
- `global.css` — Tailwind CSS 4 theme, prose typography, and copy button styles
- `astro.config.mjs` — Astro 6 config with fonts, sitemap, and MDX integrations

### How Things Connect

```
MDX page (frontmatter: layout, title, description)
  → DocsLayout.astro (sidebar, breadcrumbs, copy buttons)
    → BaseLayout.astro (meta tags, OG, canonical, GA4, JSON-LD, global CSS)
```

### Auto-Generated Files (Do Not Edit Manually)

- `dist/sitemap-index.xml` — generated by `@astrojs/sitemap` from all pages
- `dist/sitemap-0.xml` — generated by `@astrojs/sitemap`
- `dist/_astro/*.css` — compiled CSS from Tailwind + global.css

### Files That Must Be Updated Together

| When you change... | Also update... |
|---------------------|----------------|
| Add/remove a chart doc page | `DocsLayout.astro` (chartNav) + `ChartGrid.astro` |
| Add/remove a docs page | `DocsLayout.astro` (sidebarNav) |
| Add a new font | `astro.config.mjs` (fonts) + `global.css` (@theme) |
| Change site domain | `astro.config.mjs` (site) + `robots.txt` (Sitemap URL) |
| Add new structured data | `BaseLayout.astro` (JSON-LD scripts) |

## SEO Requirements

- Every page must have a unique `title` and `description` in frontmatter
- `title` renders as `{title} | HelmForge` — keep it under 60 chars
- `description` is used for `<meta>`, Open Graph, and Twitter Card — keep 50-160 chars
- Canonical URLs are auto-generated from `Astro.url`
- OG image is at `public/og-default.png` (1200x630) — referenced by all pages
- `robots.txt` in `/public` allows all crawlers and references the sitemap
- JSON-LD in `BaseLayout.astro` provides Organization and SoftwareApplication schemas

## Common Pitfalls

1. **Scoped styles breaking MDX content**: Always use `<style is:global>` for the CSS import in `BaseLayout.astro`. Without it, Astro adds `data-astro-cid-*` scoping to selectors, which breaks styles on MDX-rendered HTML and dynamically injected elements (like copy buttons).

2. **MDX frontmatter access**: In layouts used by MDX, access frontmatter via `Astro.props.frontmatter`, not `Astro.props` directly. The current code uses `Astro.props.frontmatter || Astro.props` as a fallback.

3. **Astro 6 fonts**: The `fonts` config is top-level in `astro.config.mjs`, NOT under `experimental`. This changed in Astro 6.

4. **Forgetting sidebar registration**: Creating an MDX file does NOT automatically add it to the sidebar. You must manually add it to the `sidebarNav` or `chartNav` array in `DocsLayout.astro`.

5. **Cloudflare wrangler-action**: Currently v3 (uses Node.js 20). No v4 with Node.js 24 support exists yet. The deprecation annotation is cosmetic.

6. **Vite unused import warnings**: Suppressed via `rollupOptions.onwarn` in `astro.config.mjs`. This is an Astro internal issue, not a code problem.

7. **Copy buttons**: Injected automatically by JS in `DocsLayout.astro` for all `.prose pre` elements. Do not add copy buttons manually in MDX content.
